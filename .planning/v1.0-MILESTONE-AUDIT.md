## Cross-Phase Integration Issues

### Critical: DeckBuilder → Game Flow Broken

**sessionStorage key mismatch** — DeckBuilder writes to `"forge-active-deck"` but GamePage reads `"forge-deck"`. Deck data silently lost.

**No deck loading path in WASM** — Even with fixed keys:
1. `gameStore.initGame()` receives `_deckData` but ignores it (underscore-prefixed)
2. WASM `initialize_game` accepts `_deck_data` but ignores it
3. `CardDatabase` requires filesystem access (unavailable in WASM)
4. No mechanism to instantiate cards from deck names into GameState in browser

**Missing mode parameter** — DeckBuilder navigates to `/game` without `?mode=` param, resulting in no AI opponent.

**Affected:** DECK-01, DECK-03, AI-04, PLAT-03

### Moderate: Undo Desyncs Client from Engine

`gameStore.undo()` restores client-side state from history but the WASM thread-local `GAME_STATE` retains the current state. Next `submitAction` operates on the WASM's un-reverted state, causing divergence.

**Affected:** QOL-02

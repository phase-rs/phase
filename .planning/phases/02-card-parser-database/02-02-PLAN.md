  <name>Task 1: Implement card file parser with single-card and multi-face support</name>
  <files>

  <behavior>
    Single card (Lightning Bolt):
    - Input: "Name:Lightning Bolt\nManaCost:R\nTypes:Instant\nA:SP$ DealDamage | ...\nOracle:Lightning Bolt deals 3 damage to any target.\nSVar:Picture:..."
    - Output: CardRules { layout: Single(CardFace { name: "Lightning Bolt", mana_cost: Cost{shards:[Red], generic:0}, card_type: {core_types:[Instant]}, ... }) }

    Multi-face - Adventure (Bonecrusher Giant):
    - Input contains ALTERNATE delimiter and AlternateMode:Adventure
    - Output: CardRules { layout: Adventure(main_face, adventure_face) }

    Multi-face - Transform (Nicol Bolas with DoubleFaced alias):
    - Input: AlternateMode:DoubleFaced → maps to Transform layout
    - Output: CardRules { layout: Transform(front_face, back_face) }

    Multi-face - Split (Fire // Ice):
    - Input: AlternateMode:Split
    - Output: CardRules { layout: Split(face1, face2) }

    Multi-face - Meld (Gisela):
    - Input: AlternateMode:Meld, MeldPair:Bruna
    - Output: CardRules { layout: Meld(face1, face2), meld_with: Some("Bruna") }

    Multi-face - Flip (Akki Lavarunner):
    - Input: AlternateMode:Flip
    - Output: CardRules { layout: Flip(face1, face2) }

    Multi-face - MDFC (Valki // Tibalt):
    - Input: AlternateMode:Modal
    - Output: CardRules { layout: Modal(face1, face2) }

    Edge cases:
    - Comment lines (# prefix) are skipped
    - Empty lines are skipped
    - Unknown keys are silently skipped (lenient parsing)
    - SVar:Name:Value splits on first colon to get key=SVar, then splits value on first colon to get var_name and var_value
    - PT field: "3/3" splits into power="3", toughness="3"; "*/*" into power="*", toughness="*"
    - K: line may have comma-separated keywords, each added individually
    - Colors: line provides color_override as Vec<ManaColor>
  </behavior>
  <action>
    1. Create `parser/card_parser.rs` with:

       a. `CardFaceBuilder` struct — mutable builder accumulating parsed fields during line processing:
          - All Option<T> fields for name, mana_cost, card_type, power, toughness, loyalty, defense, oracle_text, non_ability_text, flavor_name, color_override
          - Vec<String> for keywords, abilities, triggers, static_abilities, replacements
          - HashMap<String, String> for svars
          - `build(self) -> Result<CardFace, ParseError>` converts to immutable CardFace, requiring name, defaulting mana_cost to ManaCost::ZERO if not set

       b. `ParseState` struct — tracks parser state across lines:
          - `cur_face: usize` (0 = primary, 1 = alternate)
          - `alt_mode: Option<String>` (from AlternateMode line)
          - `meld_with: Option<String>` (from MeldPair line)
          - `partner_with: Option<String>` (from PartnerWith line)

       c. `pub fn parse_card_file(content: &str) -> Result<CardRules, ParseError>`:
          - Create two CardFaceBuilders (faces[0] and faces[1]) and a ParseState
          - Iterate lines: skip empty, skip lines starting with '#'
          - For each line, call `parse_line(line, &mut faces[state.cur_face], &mut state)`
          - After all lines, build CardRules from faces and state:
            - If alt_mode is None → Single(faces[0].build()?)
            - Map alt_mode string to CardLayout variant using match:
              "Split" → Split, "Flip" → Flip, "Transform"|"DoubleFaced" → Transform,
              "Meld" → Meld, "Adventure" → Adventure, "Modal" → Modal, "Omen" → Omen
            - Set meld_with from state

       d. `fn parse_line(line: &str, face: &mut CardFaceBuilder, state: &mut ParseState)`:
          - Handle bare keywords first: if line has no colon, check for "ALTERNATE" → set cur_face = 1, then return
          - Split on first ':' to get (key, value), trim value
          - Match on first byte of key, then exact key match (per CONTEXT.md decision):
            - 'A': "A" → face.abilities.push(value), "AlternateMode" → state.alt_mode = Some(value)
            - 'C': "Colors" → parse comma-separated colors into color_override
            - 'D': "Defense" → face.defense, "DeckHints"|"DeckNeeds"|"DeckHas" → skip (deferred)
            - 'F': "FlavorName" → face.flavor_name
            - 'K': "K" → split value on ',' or ';', trim each, push to keywords
            - 'L': "Loyalty" → face.loyalty
            - 'M': "ManaCost" → face.mana_cost = Some(mana_cost::parse(value)), "MeldPair" → state.meld_with
            - 'N': "Name" → face.name
            - 'O': "Oracle" → face.oracle_text
            - 'P': "PT" → split on '/', set power and toughness, "PartnerWith" → state.partner_with
            - 'R': "R" → face.replacements.push(value)
            - 'S': "S" → face.static_abilities.push(value), "SVar" → split value on first ':' again to get (var_name, var_value), insert into svars HashMap. Keys starting with "SPECIALIZE" → handle Specialize face switching
            - 'T': "T" → face.triggers.push(value), "Types" → face.card_type = Some(card_type::parse(value)), "Text" → face.non_ability_text
            - All other keys → silently skip (lenient parsing per decision)

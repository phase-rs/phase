  <name>Task 1: Add missing effect handlers (Fight, Bounce, Explore, Proliferate, CopySpell, ChooseCard)</name>
  <files>crates/engine/src/game/effects/fight.rs, crates/engine/src/game/effects/bounce.rs, crates/engine/src/game/effects/explore.rs, crates/engine/src/game/effects/proliferate.rs, crates/engine/src/game/effects/copy_spell.rs, crates/engine/src/game/effects/choose_card.rs, crates/engine/src/game/effects/mod.rs</files>
  <behavior>
    - Test: Fight between 3/3 and 2/2 results in 3 damage marked on 2/2, 2 damage on 3/3
    - Test: Bounce moves a battlefield permanent to owner's hand
    - Test: Explore with land on top puts +1/+1 counter and reveals
    - Test: Explore with nonland on top gives choice (top or graveyard) via WaitingFor
    - Test: Proliferate adds one counter to each permanent/player that already has counters
    - Test: CopySpell duplicates the top stack entry
  </behavior>
  <action>
1. **fight.rs**: `resolve()` -- get two creatures (source and target from ability.targets), each deals damage equal to its power to the other. Use existing `deal_damage::apply_damage_to_creature()` pattern. Emit DamageDealt events for both.

2. **bounce.rs**: `resolve()` -- move target permanent from battlefield to owner's hand. Read "Defined" param for target selection (Targeted, Self, etc.). Use `zones::move_to_zone(state, target_id, Zone::Hand, events)`.

3. **explore.rs**: `resolve()` -- reveal top card of library. If it's a land: put +1/+1 counter on exploring creature, keep card on top. If nonland: set WaitingFor for choice (put in hand or graveyard). Uses WaitingFor::DigChoice with keep_count=1 (reuse from Plan 03) or a simpler mechanism.

4. **proliferate.rs**: `resolve()` -- scan battlefield for permanents with counters and players with poison counters. For each, add one additional counter of each type already present. Emit CounterAdded events.

5. **copy_spell.rs**: `resolve()` -- duplicate the top stack entry. Create a new StackEntry with same ability but new ID. Push copy onto stack.

6. **choose_card.rs**: `resolve()` -- generic "choose a card" effect. Emit WaitingFor for player to select from a set of cards (e.g., ChooseCard variant or reuse DigChoice). This handles effects like "choose a card in your graveyard".

7. Register all in `effects/mod.rs`:
   ```
   registry.insert("Fight".to_string(), fight::resolve);
   registry.insert("Bounce".to_string(), bounce::resolve);
   registry.insert("Explore".to_string(), explore::resolve);
   registry.insert("Proliferate".to_string(), proliferate::resolve);
   registry.insert("CopySpell".to_string(), copy_spell::resolve);
   registry.insert("ChooseCard".to_string(), choose_card::resolve);
   ```

8. Add pub mod declarations for all new modules.
9. Write 2-3 inline tests per module.</action>

  </verify>

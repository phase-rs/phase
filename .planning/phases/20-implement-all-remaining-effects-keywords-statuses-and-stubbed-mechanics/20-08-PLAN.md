    - Test: First daybound/nightbound card entering sets it to Day
    - Test: If active player cast no spells on their turn, transitions to Night
    - Test: If active player cast 2+ spells on their turn, transitions to Day
    - Test: Daybound creatures transform when night begins
    - Test: Nightbound creatures transform when day begins
    - Test: DayTimeChanges trigger fires on day/night transition

  <action>
1. Add `day_night: Option<DayNight>` field to GameState (where `DayNight` is an enum `{ Day, Night }`). Derive Serialize/Deserialize/PartialEq/Eq/Clone/Debug.
2. Add `spells_cast_this_turn: u8` field to GameState (tracked per turn, reset at turn start).
3. Create `crates/engine/src/game/day_night.rs`:
   - `check_day_night_transition(state: &mut GameState, events: &mut Vec<GameEvent>)` -- called at end of turn (in turns.rs):
     - If day_night is None, skip.
     - If Day and active player cast 0 spells this turn -> transition to Night.
     - If Night and active player cast 2+ spells this turn -> transition to Day.
     - On transition: emit `GameEvent::DayNightChanged { new_state: DayNight }`. Transform all Daybound/Nightbound permanents accordingly (call transform::transform_permanent for each).
   - `initialize_day_night(state: &mut GameState)` -- sets day_night to Some(Day) when a daybound/nightbound card first enters the game.

5. Add `GameEvent::DayNightChanged { new_state: String }` to events.rs.
6. In `triggers.rs`: promote `DayTimeChanges` from unimplemented to real matcher that checks for DayNightChanged event.
7. Add `pub mod day_night;` to mod.rs.
8. Write 4-5 inline tests.</action>

  <action>

   - `play_face_down(state: &mut GameState, player: PlayerId, card_id: CardId, events: &mut Vec<GameEvent>) -> Result<WaitingFor, EngineError>`:
     - Move card from hand to battlefield face_down = true
     - Override characteristics: name = "" (empty), power = 2, toughness = 2, card_types = Creature only, keywords = empty, abilities = empty
     - Store original characteristics so they can be restored on turn face up
   - `turn_face_up(state: &mut GameState, player: PlayerId, object_id: ObjectId, events: &mut Vec<GameEvent>) -> Result<WaitingFor, EngineError>`:
     - Validate player controls the face-down permanent
     - Check morph cost can be paid (for now, check if the card has a morph/disguise keyword)
     - Set face_down = false, restore original characteristics
     - Emit `GameEvent::TurnedFaceUp { object_id }`
   - `manifest(state: &mut GameState, player: PlayerId, events: &mut Vec<GameEvent>) -> Result<(), EngineError>`:
     - Take top card of library
     - Put it onto battlefield face down as 2/2 creature
     - If it's a creature card, it can be turned face up by paying its mana cost

5. In triggers.rs, promote `TurnFaceUp` from unimplemented to real matcher.
6. Update `client/src/adapter/types.ts` with new action/event/GameState types.
7. Add `pub mod morph;` to mod.rs.
8. Write 4-5 inline tests.</action>

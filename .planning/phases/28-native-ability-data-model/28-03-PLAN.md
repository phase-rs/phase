  <name>Task 1: Rewrite triggers.rs for typed TriggerDefinition and ResolvedAbility</name>
  <files>crates/engine/src/game/triggers.rs</files>
  <action>
Rewrite `triggers.rs` to use typed TriggerDefinition fields instead of `params: HashMap<String, String>` and `svars: HashMap<String, String>`.

**TriggerMatcher signature**: Change from `fn(event, params: &HashMap<String, String>, source_id, state) -> bool` to `fn(event, trigger: &TriggerDefinition, source_id, state) -> bool`. Each matcher reads typed fields:
- ETB matcher: check `trigger.origin`, `trigger.destination`, `trigger.valid_card` (use matches_target_filter)
- ChangesZone matcher: same pattern
- Phase/Upkeep matchers: check `trigger.phase`
- Damage matchers: check `trigger.combat_damage`

**`process_triggers`**: Remove svars extraction. The `(controller, trigger_defs, timestamp, svars, has_prowess)` tuple drops `svars`. In the trigger match loop:
- `matcher(event, trig_def, obj_id, state)` — pass the whole TriggerDefinition instead of params
- `build_triggered_ability` no longer needs svars

**`build_triggered_ability`**: Rewrite completely. Currently looks up Execute SVar string, calls parse_ability(). New version:
```rust
fn build_triggered_ability(
    trig_def: &TriggerDefinition,
    source_id: ObjectId,
    controller: PlayerId,
) -> ResolvedAbility {
    if let Some(execute) = &trig_def.execute {
        // Pre-resolved ability definition — direct typed access
        ResolvedAbility {
            effect: execute.effect.clone(),
            targets: Vec::new(),
            source_id,
            controller,
            sub_ability: execute.sub_ability.as_ref().map(|sub| {
                Box::new(build_sub_ability(sub, source_id, controller))
            }),
        }
    } else {
        // Trigger with no execute — use a no-op or the trigger itself triggers something else
        ResolvedAbility {
            effect: Effect::Explore, // fallback no-op; adjust as needed
            targets: Vec::new(),
            source_id,
            controller,
            sub_ability: None,
        }
    }
}

fn build_sub_ability(def: &AbilityDefinition, source_id: ObjectId, controller: PlayerId) -> ResolvedAbility {
    ResolvedAbility {
        effect: def.effect.clone(),
        targets: Vec::new(),
        source_id,
        controller,
        sub_ability: def.sub_ability.as_ref().map(|sub| {
            Box::new(build_sub_ability(sub, source_id, controller))
        }),
    }
}
```

**Prowess synthetic trigger**: Update ResolvedAbility construction — no more `params: effect.to_params()` or `svars: HashMap::new()`. Just:
```rust
ResolvedAbility {
    effect: prowess_effect,
    targets: vec![TargetRef::Object(obj_id)],
    source_id: obj_id,
    controller,
    sub_ability: None,
}
```

**Update all trigger matcher functions** in the registry to read typed fields instead of params.get("key").

**Update tests**: Rewrite any test TriggerDefinition construction to use typed fields instead of HashMap.

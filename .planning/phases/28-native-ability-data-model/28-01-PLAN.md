<objective>
Add typed struct fields to all definition types and create the migration binary.

Purpose: This plan adds typed fields alongside the existing `params: HashMap<String, String>` on TriggerDefinition, StaticDefinition, ReplacementDefinition, and adds typed condition/sub-ability fields to AbilityDefinition's remaining_params. It also replaces `Effect::Other` with `Effect::Unimplemented` and writes a migration binary that converts old-format JSON files to the new typed format by resolving SVar chains at migration time. The typed fields are additive -- `params` stays temporarily with `#[serde(default)]` so both old and new JSON formats deserialize. Plan 02 will switch runtime consumers to the typed fields and remove `params`.

Output: Updated type definitions, migration binary source
</objective>

<task type="auto">
  <name>Task 2: Write migration binary to convert old-format JSON to typed format</name>
  <files>crates/engine/src/bin/migrate_typed.rs, crates/engine/Cargo.toml</files>
  <action>
Create `crates/engine/src/bin/migrate_typed.rs` as a binary gated behind `forge-compat` feature (add `[[bin]] name = "migrate-typed" required-features = ["forge-compat"]` to engine's Cargo.toml).

The migration binary:

1. **Reads** each `data/abilities/*.json` file as old-format `AbilityFile` (with `params` HashMaps on definitions)
2. **For each TriggerDefinition**: Extracts `Origin`, `Destination`, `ValidCard`, `ValidSource`, `ValidActivatingPlayer`, `ValidPlayer`, `ValidToken`, `CounterType`, `DamageAmount`, `CombatDamage`, `Phase`, `TriggerZones`, `TriggerDescription` from `params` into the new typed fields. For `Execute` param: looks up the SVar name in the card's `svars` map, calls `parse_ability()` on the SVar string, and stores the result in `execute: Some(Box::new(ability_def))`. Track visited SVar names to prevent cycles (depth limit 10, matching `MAX_CHAIN_DEPTH`). After extraction, clear the consumed keys from `params`.
3. **For each StaticDefinition**: Extracts `Affected`, `AddPower` (parse to i32), `AddToughness`, `SetPower`, `SetToughness`, `AddKeyword`, `RemoveKeyword`, `AddType`, `RemoveType`, `SetColor`, `AddColor`, `AddAbility`, `RemoveAllAbilities` (parse to bool), `CheckSVar`, `SVarCompare`, `Cost`, `Target`, `Description` from `params`. After extraction, clear consumed keys.
4. **For each ReplacementDefinition**: Extracts `ActiveZones`, `Origin$`, `Destination$`, `NewDestination$`, `ValidCard`, `ValidSource`, `ValidPlayer`, `Prevent`/`Exile`/`Double` (parse "True" to bool), `NewAmount$`, `NewCount$`, `NewName$`, `DamageType$`, `Layer`, `Description` from `params`. For `Execute` param: resolve SVar like triggers. Clear consumed keys.
5. **For each AbilityDefinition**: Extracts `ConditionCompare`, `ConditionPresent`, `ConditionZone`, `ConditionSVarCompare`, `Defined` from `remaining_params`. For `SubAbility`/`Execute` in `remaining_params`: resolve SVar chain and populate `sub_ability` field. Replaces `Effect::Other { api_type, params }` with `Effect::Unimplemented { name: api_type }`. Clear consumed keys from `remaining_params`.
6. **For each card**: After all definitions are migrated, clear the top-level `svars` field (set to empty HashMap).
7. **Writes** the migrated JSON back to the same file path (overwrite in place).
8. **Reports** statistics: total files processed, SVars resolved, unresolved SVars (with card names), remaining non-empty params (should be zero for runtime-consumed keys).

Use `rayon::par_iter` for parallel processing of the 32K files.

Usage: `cargo run --bin migrate-typed --features forge-compat -- data/abilities/`

Handle edge cases:
- Missing SVar references: set `execute: None` and log a warning (don't fail)
- SVar chain cycles: stop at depth 10 with a warning
- Parse failures in `parse_ability()`: leave execute as None, log card name + SVar
- `params` keys not in the extraction list: leave them in `params` (they're metadata the engine doesn't consume; Plan 02 will decide whether to drop them)

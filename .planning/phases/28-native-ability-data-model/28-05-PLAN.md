**Step 2: Update `client/src/adapter/types.ts`** based on compilation errors from the new types:

Key breaking changes to address:
- **Remove `svars: Record<string, string>`** from any GameObject type definition (approximately line 101). The field no longer exists on the Rust side.
- **`keywords` type change**: Was `string[]`, now a typed Keyword union from tsify. Update any code that constructs or reads keywords as strings to use the typed format.
- **`power`/`toughness` type change**: Was `string | null`, now `PtValue | null` (where PtValue is `{ type: "Fixed", value: number } | { type: "Variable", value: string }`). Update any code that reads P/T.
- **Effect variants change shape**: If the frontend inspects Effect objects (for logging, animation, etc.), update the discriminated union types.
- **TargetSpec -> TargetFilter**: If the frontend references TargetSpec anywhere, update to TargetFilter.

**Step 3: Fix TypeScript compilation errors:**
Run `pnpm run type-check` from client/ and fix any errors. Typical fixes:
- Remove references to `.svars` on game objects
- Update keyword comparison code (if any component checks `keywords.includes("Flying")` it needs to match the typed Keyword shape)
- Update P/T display code to handle PtValue objects instead of strings

**Step 4: Update test mocks** in client test files:
Any test that constructs mock game state objects with `svars: {}` must remove that field. Any mock with `keywords: ["Flying"]` must use the typed format.

**Step 5: Run frontend tests:**
```bash
cd client && pnpm test -- --run
```

**Step 6: Run full lint + type-check:**

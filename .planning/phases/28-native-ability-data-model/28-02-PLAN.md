<objective>
Migrate all runtime consumers from HashMap string lookups to typed field access and remove all Forge-era string infrastructure.

Purpose: This is the core transformation. Every file that reads `def.params.get("X")` or `ability.svars.get("Y")` switches to reading typed struct fields. After this plan, zero runtime code accesses HashMap params or SVars -- all string-keyed lookups are eliminated. The `params` field is removed from all definition structs, `svars` is removed from CardFace and GameObject, `remaining_params` is removed from AbilityDefinition, and `parse_ability()` is gated behind `forge-compat`.

Output: Fully typed runtime with no HashMap params, no svars, no runtime string parsing
</objective>

**`crates/engine/src/game/triggers.rs`:**

- Remove all `svars` access from this file. The trigger's `execute` field already contains the fully typed ability definition.
- Update `build_trigger_registry()` and all matcher functions to use the new signature.

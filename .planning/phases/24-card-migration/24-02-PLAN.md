<task type="auto">
  <name>Task 2: Build Forge vs JSON parity integration tests</name>
  <files>crates/engine/tests/parity.rs</files>
  <action>
Create `crates/engine/tests/parity.rs` as an integration test module. Run via `cargo test --test parity`.

The test structure:

1. **Helper functions**:
   - `data_dir() -> PathBuf`: Returns `Path::new(env!("CARGO_MANIFEST_DIR")).join("../../data")`
   - `load_manifest() -> Vec<String>`: Reads `data/standard-cards.txt`, filters comments and blank lines, returns card names
   - `compare_card_rules(name: &str, forge: &CardRules, json: &CardRules) -> Vec<String>`: Compares the two structurally, returns a list of mismatch descriptions (empty = match). For each face in the layout, compare:
     - `name` (exact match)
     - `mana_cost` (via Debug format comparison -- ManaCost doesn't impl PartialEq directly, use serde_json::to_value comparison)
     - `card_type` (via Debug format)
     - `power`, `toughness`, `loyalty` (direct comparison)
     - `keywords` length and contents (sorted comparison since order may differ)
     - `abilities` count and per-ability: `kind`, `effect` (compare via serde_json::to_value for Effect enum), `cost` (compare via serde_json::to_value)
     - `triggers` count and per-trigger: `mode` (via Debug)
     - `statics` count and per-static: `mode` (via Debug)
     - `replacements` count
   - For `Effect::Other` fields, compare `api_type` and `params` HashMap (the round-trip should preserve these)
   - Do NOT compare `svars` (JSON path doesn't populate these) or `scryfall_oracle_id` (only on JSON path) or `oracle_text` / `non_ability_text` (only from MTGJSON)

2. **Explicit divergence allowlist**: Define a constant `ALLOWED_DIVERGENCES` that maps card names to expected divergence descriptions. The JSON path synthesizes implicit abilities that the Forge path derives at runtime. The allowlist MUST be limited to these specific categories:
   - **Basic land mana abilities**: JSON-loaded basic lands (Plains, Island, Swamp, Mountain, Forest) have an explicit mana-producing `AbilityDefinition` that the Forge path synthesizes at runtime. Allow `abilities` count difference of exactly 1 for basic lands where the extra ability is a mana ability.
   - **Equipment equip abilities**: JSON-loaded equipment cards have an explicit `Equip` activated ability that the Forge path derives from the `Equip` keyword at runtime. Allow `abilities` count difference of exactly 1 for equipment where the extra ability has `AbilityKind::Activated` with an equip cost.
   - **No other divergences are allowed.** If `compare_card_rules()` returns mismatches for a card NOT in the allowlist, or mismatches that don't match the allowlist's expected pattern, the test MUST fail.

3. **Positive assertions for implicit abilities**: In addition to the parity comparison, add explicit positive checks:
   - `assert_basic_lands_have_mana_ability()`: For each basic land in the manifest (if any), assert that the JSON-loaded version has at least one ability with a mana-producing effect. If a basic land loaded via JSON has zero mana abilities, fail with a clear message.
   - `assert_equipment_has_equip_ability()`: For each equipment card in the manifest (if any), assert that the JSON-loaded version has at least one activated ability with an equip-related cost/effect. If an equipment card loaded via JSON has no equip ability, fail with a clear message.
   - These assertions run regardless of the parity comparison -- they are independent guards against the JSON path silently dropping implicit abilities.

4. **Main test**: `parity_all_standard_cards()`:
   - Load manifest
   - Load Forge DB via `CardDatabase::load(&data_dir().join("standard-cards"))`
   - Load JSON DB via `CardDatabase::load_json(&data_dir().join("mtgjson/test_fixture.json"), &data_dir().join("abilities"))`
   - For each card name in manifest:
     - Look up in both DBs (panic with clear message if missing)
     - Call `compare_card_rules()`
     - Filter out allowed divergences for this card
     - Collect remaining mismatches as failures
   - Run `assert_basic_lands_have_mana_ability()` and `assert_equipment_has_equip_ability()` against the JSON DB
   - Report all failures at end (don't stop at first mismatch)
   - Assert zero unexpected mismatches

5. **Individual card spot-checks**: Add a few targeted tests for cards known to exercise specific patterns:
   - `parity_lightning_bolt()` -- simple instant, Spell ability with DealDamage effect
   - `parity_jace_the_mind_sculptor()` -- planeswalker with loyalty costs
   - `parity_lovestruck_beast()` -- adventure multi-face card

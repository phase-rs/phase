<task type="auto">
  <name>Task 1: Add --json mode to coverage report binary</name>
  <files>crates/engine/src/bin/coverage_report.rs</files>
  <action>
Modify `crates/engine/src/bin/coverage_report.rs` to support a `--json` flag that loads cards via `CardDatabase::load_json()` instead of `CardDatabase::load()`.

Current argument parsing extracts `--ci` and a positional path. Extend it to also check for `--json`:

1. **New flag `--json`**: When present, load cards via JSON path instead of Forge path.

2. **JSON mode loading**: When `--json` is set:
   - Read the Standard card manifest from `data/standard-cards.txt` (resolve relative to the positional path argument, or use a sensible default -- the manifest is at the repo root)
   - Load via `CardDatabase::load_json(mtgjson_path, abilities_dir)` where:
     - `mtgjson_path` = `data/mtgjson/test_fixture.json` (relative to working dir or derived from positional arg)
     - `abilities_dir` = `data/abilities/`
   - The positional argument in `--json` mode can be the data root directory (e.g., `data/`) from which mtgjson/ and abilities/ subdirectories are resolved

3. **Forge mode (unchanged)**: When `--json` is NOT set, behavior is identical to current: `CardDatabase::load(&path)`.

4. **Manifest filtering**: In JSON mode, `CardDatabase::load_json()` loads ALL cards in the abilities directory (~32,300). The coverage report must filter to only analyze cards listed in `data/standard-cards.txt`. Implement this filtering directly within `coverage_report.rs`:
   - Load the full JSON database
   - Read the manifest file (`data/standard-cards.txt`), filter comments and blank lines
   - Build a `HashSet<String>` of manifest card names
   - After calling `analyze_standard_coverage(&db)`, filter the `CoverageSummary` results to only include cards from the manifest set
   - Alternatively, iterate the manifest names, look up each in the DB, and run the existing `has_unimplemented_mechanics()` check per card to build a filtered `CoverageSummary`
   - Do NOT modify `crates/engine/src/game/coverage.rs` -- keep all filtering logic in the binary itself

5. **Usage examples**:
   ```
   # Forge mode (existing, unchanged):
   cargo run --bin coverage-report -- data/standard-cards/ --ci

   # JSON mode (new):
   cargo run --bin coverage-report -- --json data/ --ci
   ```

6. **Help text**: Update the usage message to document the --json flag.

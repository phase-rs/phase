  <action>
    For each mutation site listed in the research's "Mutation Site Inventory", wrap the actual mutation in a replace_event() call. The pattern for each site:

    ```rust
    let proposed = ProposedEvent::Variant { /* fields */, applied: HashSet::new() };
    match replacement::replace_event(state, proposed, events) {
        ReplacementResult::Execute(event) => {
            // Extract (possibly modified) fields from event, perform the actual mutation
        }
        ReplacementResult::Prevented => {
            // Skip the mutation entirely
        }
        ReplacementResult::NeedsChoice(player) => {
            // Store context, set state.waiting_for = WaitingFor::ReplacementChoice
            // Return early -- engine will resume after player chooses
            return Ok(()); // or appropriate early return
        }
    }
    ```

    **Specific mutation sites to hook:**

    1. **deal_damage.rs** (resolve fn): Before applying damage to creature (damage_marked +=) or player (life -=), create ProposedEvent::Damage. On Execute, apply the (possibly modified) amount. On Prevented, skip damage entirely.

    2. **draw.rs** (resolve fn): Before drawing cards, create ProposedEvent::Draw. On Execute, draw the (possibly modified) count. On Prevented, skip draw.

    3. **life.rs** (resolve_gain): Before adding life, create ProposedEvent::LifeGain. On Execute, add the (possibly modified) amount. (resolve_lose): Before subtracting life, create ProposedEvent::LifeLoss.

    4. **change_zone.rs** (resolve fn): Before calling move_to_zone, create ProposedEvent::ZoneChange. On Execute, use the (possibly modified) destination zone.

    5. **destroy.rs** (resolve fn): Before moving to graveyard, create ProposedEvent::Destroy. On Execute, perform the destruction. On Prevented, skip. The destruction itself also calls move_to_zone which will create its own ZoneChange proposal (nested replacement -- depth cap handles this).

    6. **counters.rs** (resolve_add, resolve_remove): Before modifying counters, create ProposedEvent::AddCounter / RemoveCounter. On Execute, apply the (possibly modified) count. Set `state.layers_dirty = true` for +1/+1 and -1/-1 counters.

    7. **token.rs** (resolve fn): Before creating the token object, create ProposedEvent::CreateToken. On Execute, create the token. On Prevented, skip.

    8. **discard.rs** (resolve fn): Before moving card from hand to graveyard, create ProposedEvent::Discard. On Execute, perform the discard (with possibly modified destination).

    9. **tap_untap.rs** (resolve_tap, resolve_untap): Before toggling tapped state, create ProposedEvent::Tap / Untap. On Prevented, skip.

    10. **sacrifice.rs** (resolve fn): Before moving permanent to graveyard, create ProposedEvent::Sacrifice. On Execute, perform the sacrifice.

    11. **zones.rs** (move_to_zone): This is the low-level zone transfer used by multiple callers. Add an optional `replace` parameter or create a `move_to_zone_with_replacement` wrapper. Effect handlers call the replacement-aware version; internal engine operations (like stack resolution moving spell to battlefield) can call the raw version when replacement is already handled at a higher level. Set `state.layers_dirty = true` when zone is Battlefield (entering or leaving).

    12. **combat_damage.rs**: Before applying each point of combat damage, create ProposedEvent::Damage with `is_combat: true`. Use the same pattern as deal_damage.rs.

    13. **turns.rs** (untap step): Before untapping each permanent in untap step, create ProposedEvent::Untap. Honor Prevented (for "doesn't untap during untap step" effects).

    **Important:** For NeedsChoice returns in effect handlers, the handler must store enough context for the engine to resume. Since the replacement pipeline stores PendingReplacement on GameState, and the engine's apply() already handles ReplacementChoice continuation, the effect handler can simply return Ok(()) after setting waiting_for. The engine loop will handle continuation when the player responds.

    **Mark layers_dirty:** Any mutation that changes the battlefield (zone changes to/from battlefield, counter modifications) should set `state.layers_dirty = true`.

    **Wire evaluate_layers into engine checkpoint flow:**

    1. **sba.rs**: At the start of the SBA check function (before the fixpoint loop), add:
    ```rust
    if state.layers_dirty {
        layers::evaluate_layers(state);
        // layers_dirty is cleared inside evaluate_layers
    }
    ```
    This ensures computed P/T is current before checking 0-toughness, lethal damage, etc.

    2. **engine.rs**: In the main apply() function, after SBA + trigger processing, if layers_dirty, call evaluate_layers. This catches any state changes that didn't go through SBAs (edge cases).

    3. **casting.rs** (or wherever target legality is checked): Before target legality validation, ensure layers are evaluated. Add `if state.layers_dirty { layers::evaluate_layers(state); }` before target checks that depend on characteristics (type-based targeting, power/toughness comparisons).

    **Integration tests** (add to a new test file or at the bottom of engine.rs tests):

    1. `test_replacement_redirects_destruction_to_exile`:
       - Set up: Create a creature on battlefield with a ReplacementDefinition { event: "Moved", params: { Origin$: "Battlefield", Destination$: "Graveyard", ReplaceDestination$: "Exile" } } on the creature itself (self-replacement).
       - Action: Cast a Destroy effect targeting the creature.
       - Verify: Creature ends up in exile (not graveyard). ReplacementApplied event was emitted.

    2. `test_layer_evaluation_before_sba`:
       - Set up: Create a 2/2 creature on battlefield. Create a lord that gives all creatures -2/-2 via static ability (Continuous mode, AddPower$ -2, AddToughness$ -2).
       - Action: Pass priority to trigger SBA processing.
       - Verify: evaluate_layers runs, creature's computed toughness becomes 0, SBA destroys it.

    3. `test_combat_damage_through_replacement`:
       - Set up: Create an attacking creature and a DamageDone replacement that prevents all combat damage.
       - Action: Resolve combat damage step.
       - Verify: No damage applied to defending player (replacement prevented it).

    4. `test_lord_buff_applies_and_unapplies`:
       - Set up: Create a lord with +1/+1 buff and a 2/2 bear. Verify bear is 3/3 after layer eval.
       - Action: Destroy the lord (move to graveyard).
       - Verify: After next layer evaluation, bear returns to 2/2.

    5. `test_full_pipeline_destroy_with_replacement_and_trigger`:
       - Set up: Create creature A with "if would die, exile instead" replacement. Create creature B with "when a creature is exiled, draw a card" trigger.
       - Action: Destroy creature A.
       - Verify: A goes to exile (not graveyard). B's trigger fires. Controller draws a card.
       - This proves replacement -> mutation -> trigger pipeline works end-to-end.

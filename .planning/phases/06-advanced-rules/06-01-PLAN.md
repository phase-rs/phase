  <action>
    Replace the stub matchers and appliers in the replacement registry with real implementations. All handler logic lives inline in replacement.rs (not split into submodules -- keep it simple until the file gets unwieldy).

    **Handler implementations** (14 handlers matching Forge's ReplacementType variants):

    For each handler, implement a `matches` function and an `apply` function:

    1. **Moved** (ZoneChange): Match when ProposedEvent::ZoneChange and params match origin/destination zones (Origin$, Destination$ params). Apply by changing the `to` field of ZoneChange variant. This is the most important handler -- covers "dies -> exile instead" patterns.

    2. **DamageDone**: Match ProposedEvent::Damage. Apply by modifying amount (for prevention) or redirecting target. Params: ValidTgts$ for source filter, DamageType$ for combat/noncombat filter.

    3. **Destroy**: Match ProposedEvent::Destroy. Apply by preventing destruction (indestructible-like effects) or changing to exile.

    4. **Draw**: Match ProposedEvent::Draw. Apply by modifying count or preventing draw entirely (replacement returns Prevented for "skip your draw" effects).

    5. **GainLife**: Match ProposedEvent::LifeGain. Apply by modifying amount or preventing.

    6. **LifeReduced**: Match ProposedEvent::LifeLoss. Apply by modifying amount or preventing.

    7. **AddCounter**: Match ProposedEvent::AddCounter. Apply by modifying count or counter type (e.g., Doubling Season doubles counters).

    8. **RemoveCounter**: Match ProposedEvent::RemoveCounter. Apply by modifying count.

    9. **CreateToken**: Match ProposedEvent::CreateToken. Apply by modifying token properties or preventing.

    10. **Discard**: Match ProposedEvent::Discard. Apply by redirecting (discard -> exile).

    11. **Tap**: Match ProposedEvent::Tap. Apply by preventing tap.

    12. **Untap**: Match ProposedEvent::Untap. Apply by preventing untap.

    13. **Sacrifice**: Match ProposedEvent::Sacrifice. Apply by preventing or redirecting.

    14. **Counter** (spell countering): Match ProposedEvent::ZoneChange where from=Stack (countered spell going to graveyard). Apply by changing destination (e.g., "can't be countered" or "countered spell goes to exile").

    Each matcher checks: (a) event variant matches handler's expected type, (b) handler params (from ReplacementDefinition) match event specifics (e.g., zone filters, creature type filters). Use `card_matches_filter` helper from triggers module where applicable for source/target filtering.

    Each applier: (a) modifies the ProposedEvent variant's mutable fields as specified by params, (b) for prevention effects returns `ApplyResult::Prevented`, (c) for modification effects returns `ApplyResult::Modified(event)`.

    **Pipeline tests** (in #[cfg(test)] mod tests at bottom of replacement.rs):

    1. `test_single_replacement_zone_change`: Create a creature on battlefield with a Moved replacement (death -> exile). Propose a ZoneChange(Battlefield -> Graveyard). Verify replace_event returns Execute with destination changed to Exile.

    2. `test_once_per_event_enforcement`: Same setup as above but with two Moved replacements on the same object. Verify the second one is considered a separate candidate (different ReplacementId index) and both can apply, but each only once.

    3. `test_multiple_replacements_needs_choice`: Create two objects each with a Moved replacement that could apply to the same event. Verify replace_event returns NeedsChoice with the affected player.

    4. `test_continue_replacement_after_choice`: After NeedsChoice, call continue_replacement with chosen index. Verify it applies the chosen replacement and re-evaluates. If no more candidates, returns Execute.

    5. `test_depth_cap`: Create a scenario that would loop (though once-per-event should prevent it). Verify depth cap of 16 is respected.

    6. `test_damage_prevention`: Create an object with DamageDone replacement that prevents damage. Verify Damage event is replaced with Prevented.

    7. `test_no_replacements_passthrough`: Propose an event with no matching replacements on battlefield. Verify replace_event returns Execute with the original event unmodified.

<task type="auto" tdd="true">
  <name>Task 2: Zone transfer operations</name>
  <files>
    crates/engine/src/game/zones.rs
    crates/engine/src/game/mod.rs
  </files>
  <behavior>
    - move_to_zone(state, object_id, destination_zone, events) correctly removes from source zone and adds to destination
    - Zone transfer generates a ZoneChanged event with from/to zones
    - Moving to library can specify top or bottom position
    - Object's zone field is updated after transfer
    - Transferring from hand to battlefield works (for land plays)
    - Transferring from library to hand works (for draws)
    - Transferring from battlefield to graveyard works (for creature death)
    - add_object_to_zone helper adds ObjectId to the correct zone collection (player-specific or shared)
    - remove_from_zone helper removes ObjectId from the correct zone collection
    - create_object(state, ...) assigns next_object_id and inserts into objects map
  </behavior>
  <action>
    Create `crates/engine/src/game/zones.rs` with:

    1. `create_object(state: &mut GameState, card_id: CardId, owner: PlayerId, name: String, zone: Zone) -> ObjectId` — allocates next_object_id, creates GameObject with defaults, inserts into state.objects, adds to zone.

    2. `move_to_zone(state: &mut GameState, object_id: ObjectId, to: Zone, events: &mut Vec<GameEvent>)` — reads object's current zone, calls remove_from_zone for source, calls add_to_zone for destination, updates object.zone field, pushes ZoneChanged event. Follows the pattern from RESEARCH.md.

    3. `remove_from_zone(state: &mut GameState, object_id: ObjectId, zone: Zone, owner: PlayerId)` — removes ObjectId from the appropriate zone collection. For player zones (Library, Hand, Graveyard), removes from the owning player's collection. For shared zones (Battlefield, Stack, Exile), removes from state-level collection. Use `retain(|id| *id != object_id)`.

    4. `add_to_zone(state: &mut GameState, object_id: ObjectId, zone: Zone, owner: PlayerId)` — adds ObjectId to the appropriate zone collection.

    5. `move_to_library_position(state: &mut GameState, object_id: ObjectId, owner: PlayerId, top: bool, events: &mut Vec<GameEvent>)` — like move_to_zone but places at index 0 (top) or pushes to end (bottom) of library. Convention: library[0] = top of library.

    Update `crates/engine/src/game/mod.rs` to include `pub mod zones;` and re-export public functions.

    Write tests in the zones.rs module covering each transfer scenario. Create a test helper that sets up a minimal GameState with one object.
  </action>

    <automated>cargo test -p engine game::zones -- --nocapture 2>&1 | tail -20</automated>
  </verify>
  <done>
    - Objects can be created and placed in any zone
    - Objects can be moved between any combination of zones
    - Zone transfers generate ZoneChanged events
    - Player zones (library/hand/graveyard) are correctly associated with owning player
    - Shared zones (battlefield/exile) work for objects from any player
    - Library positioning (top/bottom) works correctly
  </done>

  <action>
**ManaPaymentUI.tsx** — Complete rewrite with smart auto-pay logic:

**Cost inference:** When `WaitingFor::ManaPayment` is received, determine the cost being paid:
1. Check `gameState.stack` — if non-empty, the top entry's `source_id` maps to `gameState.objects[source_id].mana_cost`
2. If stack is empty (rare for ManaPayment), fall back to auto-pay

**Ambiguity detection:** Parse the cost's shards array. A cost is "ambiguous" if any shard contains "/" (hybrid or phyrexian) OR if any shard is "X". Create a helper `hasAmbiguousCost(shards: string[]): boolean` that checks `shard.includes('/')` or `shard === 'X'`.

**Auto-pay for simple costs:** If `!hasAmbiguousCost(shards)`, immediately dispatch `PassPriority` via `useEffect` (the engine handles optimal mana source selection). Do NOT render any UI.

**Ambiguous cost UI** — When UI is shown, render in a fixed bottom-center modal (same as current positioning):
- Title: "Pay Mana Cost" with the card name
- Cost display row: Render each shard using `ManaSymbol` component
- For each shard, determine interaction:

**X cost handling:**
- Show a horizontal `<input type="range">` slider from 0 to max-affordable-X
- Max-affordable-X = total mana in pool minus non-X colored requirements
- Display "X={value}: Pay {total}" label updating in real-time
- Store X value in local state

**Phyrexian shard handling:**
- Each phyrexian shard (e.g. "W/P") shows as a toggleable button
- Default: pay mana (ManaSymbol with mana color). Toggle: pay life (heart icon with "2 life" label)
- Show total life cost at bottom if any phyrexian toggled to life payment
- Store choices in local `Map<number, 'mana' | 'life'>` keyed by shard index

**Hybrid shard handling:**
- Each hybrid shard (e.g. "W/U") shows as a toggleable button
- Tap to switch between color options (e.g. W or U)
- Store choices in local `Map<number, string>` keyed by shard index

**Confirm button:** "Pay" button (emerald tone via gameButtonClass) dispatches PassPriority (engine handles the actual payment with the choices — for v1.1 the engine auto-pays, so the UI is informational + the choices help the player understand what they're paying).

Use `AnimatePresence` and `motion.div` for smooth entry/exit transitions. Use `gameButtonClass` for the confirm button.

**ManaBadge.tsx** — Read the existing file first. Add a subtle glow ring on non-zero mana amounts (`ring-1 ring-{color}/30` where color matches the mana type). Increase size slightly for better visibility.

  <name>Task 1: Refactor AnimationOverlay to step-based processing with all VFX</name>
  <files>client/src/components/animation/AnimationOverlay.tsx, client/src/pages/GamePage.tsx</files>
  <action>
    Rewrite `AnimationOverlay.tsx` to process AnimationSteps instead of flat AnimationEffects:

    Props: `containerRef: RefObject<HTMLElement>` (for screen shake target, passed from GamePage)

    State/refs:
    - `particleRef` for ParticleCanvas handle
    - `activeFloats` state for floating numbers (keep existing pattern)
    - `activeTurnBanner` state: `{ turnNumber, isPlayerTurn } | null`
    - `activeVignette` state: `{ damageAmount } | null`
    - `activeReveals` state for card reveal bursts
    - Read `vfxQuality` and `animationSpeed` from preferencesStore (use selectors)
    - Compute `speedMultiplier` from `SPEED_MULTIPLIERS[animationSpeed]`

    Step processing loop (replaces old effect-by-effect processing):
    - Subscribe to animationStore `steps` and `isPlaying`
    - When isPlaying and steps exist, call `playNextStep()` to get next step
    - For each step, process ALL effects in parallel:
      - `DamageDealt`: emit particle burst at target position (from snapshot or positionRegistry), create floating number with damage amount. If vfxQuality is 'full', determine shake intensity from damage amount (1-3=light, 4-6=medium, 7+=heavy) and call `applyScreenShake`. If target is Player, activate DamageVignette.
      - `LifeChanged`: create floating number at player HUD position (same logic as current)
      - `CreatureDestroyed`: emit red/dark particle burst at creature's snapshot position. Create death clone: a `motion.div` at snapshot position with the card image, `exit={{ opacity: 0, scale: 0.8 }}` over ~400ms. Use AnimatePresence for exit animation. Key namespace: `death-${objectId}`.
      - `PermanentSacrificed`: same as CreatureDestroyed
      - `SpellCast`: emit WUBRG particle burst at card position
      - `AttackersDeclared`: emit white particle bursts on all attackers
      - `BlockersDeclared`: no particle burst needed (block lines handled by BlockerArrow component)
      - `TurnStarted`: set activeTurnBanner state (auto-clears after animation)
      - `ZoneChanged` where `to === 'Battlefield'`: trigger CardRevealBurst at card position
      - `TokenCreated`: trigger CardRevealBurst at card position
    - After processing all effects in the step, wait for `step.duration * speedMultiplier` ms
    - Then process the next step
    - When all steps are done, clear active states

    Death clone implementation (per research recommendation — React portal approach):
    - Maintain `activeDeathClones` state: `Array<{ id: number; position: DOMRect; cardName: string }>`
    - When CreatureDestroyed effect processed, look up card name from gameStore.gameState.objects (state hasn't been updated yet due to snapshot-before-dispatch), add to activeDeathClones with snapshot position
    - Render death clones as absolutely-positioned card images in an overlay div (z-index: 45, between board and particles)
    - Use AnimatePresence + motion.div with exit animation: fade to 0 opacity + scale 0.8 over ~400ms
    - Remove from activeDeathClones array after exit animation completes (onAnimationComplete)
    - Simultaneous deaths (board wipe): all clones added in the same step, all animate at once (per user decision)

    Render structure:
    ```
    <>
      {/* Death clones overlay (z-45) */}
      <div style={{ position: 'fixed', inset: 0, pointerEvents: 'none', zIndex: 45 }}>
        <AnimatePresence>
          {activeDeathClones.map(clone => <DeathClone key={`death-${clone.id}`} ... />)}
        </AnimatePresence>
      </div>

      {/* Damage vignette (z-45) */}
      <AnimatePresence>
        {activeVignette && <DamageVignette ... />}
      </AnimatePresence>

      {/* Turn banner (z-50) */}
      <AnimatePresence>
        {activeTurnBanner && <TurnBanner ... />}
      </AnimatePresence>

      {/* Card reveals */}
      <AnimatePresence>
        {activeReveals.map(reveal => <CardRevealBurst ... />)}
      </AnimatePresence>

      {/* Particles (z-55) */}
      <ParticleCanvas ref={particleRef} />

      {/* Floating numbers (z-60) */}
      <AnimatePresence>
        {activeFloats.map(f => <FloatingNumber ... speedMultiplier={speedMultiplier} />)}
      </AnimatePresence>
    </>
    ```

    Update `GamePage.tsx`:
    - Add `useRef<HTMLDivElement>(null)` for the root game container div
    - Attach ref to the root div: `<div ref={containerRef} className="relative h-screen w-screen ...">`
    - Pass to AnimationOverlay: `<AnimationOverlay containerRef={containerRef} />`
  </action>

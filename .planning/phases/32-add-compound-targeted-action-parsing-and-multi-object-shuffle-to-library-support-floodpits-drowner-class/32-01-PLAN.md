    **1. parser/oracle_effect.rs — Create try_split_compound function:**
    Following the pattern of try_split_pump_compound (line ~3071), create:
    ```rust
    /// Generic compound effect splitter for "verb target X and [additional effect on it]" patterns.
    /// Called after a primary targeted effect is parsed, when remainder text exists.
    /// Splits at first " and " or ", then " connector and parses the remainder as an effect chain.
    /// If the remainder references "it"/"that creature"/"that player", emits ParentTarget.
    fn try_split_compound(remainder: &str) -> Option<AbilityDefinition> {
        let lower = remainder.to_lowercase();
        // Find connector: " and " or ", then "
        let (connector_pos, connector_len) = if let Some(pos) = lower.find(" and ") {
            (pos, 5)
        } else if let Some(pos) = lower.find(", then ") {
            (pos, 7)
        } else {
            return None;
        };
        let after_connector = remainder[connector_pos + connector_len..].trim();
        if after_connector.is_empty() {
            return None;
        }
        // Parse remainder as effect. contains_object_pronoun detects "it"/"that creature"
        // which maps to ParentTarget for the sub_ability's target
        let sub = parse_effect_chain(after_connector, AbilityKind::Spell);
        Some(sub)
    }
    ```
    The exact implementation should follow the idioms of try_split_pump_compound. The key insight: if the remainder's parsed effect has target=Any or target=None, and the original text contained "it"/"that creature" (via contains_object_pronoun), replace the target with TargetFilter::ParentTarget.

  <action>
Create `crates/engine/src/game/effects/` directory and module.

**mod.rs** -- Effect registry and dispatch:
- Define `type EffectHandler = fn(state: &mut GameState, ability: &ResolvedAbility, events: &mut Vec<GameEvent>) -> Result<(), EffectError>;`
- `pub fn build_registry() -> HashMap<String, EffectHandler>` -- inserts all 15 handlers
- `pub fn resolve_effect(registry: &HashMap<String, EffectHandler>, state: &mut GameState, ability: &ResolvedAbility, events: &mut Vec<GameEvent>) -> Result<(), EffectError>` -- looks up api_type in registry, calls handler, returns Unregistered error if not found
- Declare all sub-modules: `pub mod draw; pub mod deal_damage;` etc.

**15 handler files** -- each exports `pub fn resolve(state, ability, events) -> Result<(), EffectError>`:

1. **draw.rs**: Read `NumCards` param (default 1). Draw cards for ability.controller by calling `zones::move_to_zone` from top of library to hand. Emit CardDrawn events.

2. **deal_damage.rs**: Read `NumDmg` param. For each target: if Object, increment damage_marked on the GameObject; if Player, decrement life. Emit DamageDealt events.

3. **change_zone.rs**: Read `Origin` and `Destination` params. Move target objects between zones using `zones::move_to_zone`. Handle "Battlefield", "Hand", "Graveyard", "Library", "Exile" string values mapped to Zone enum.

4. **pump.rs**: Read `NumAtt` and `NumDef` params (default "0"). For each target Object, add values to power and toughness (using i32 parse). Note: this is temporary pump for the current resolution -- permanent pump tracking is Phase 6.

5. **destroy.rs**: For each target Object on battlefield, move to owner's graveyard via `zones::move_to_zone`. Check for indestructible keyword (skip if present). Emit CreatureDestroyed events.

6. **counter.rs**: For each target Object on the stack, find the StackEntry by id, remove it from state.stack, move the card to graveyard via `zones::move_to_zone`. Emit SpellCountered events.

7. **token.rs**: Read `Name`, `Power`, `Toughness`, `Types`, `Colors` params. Create a new GameObject via `zones::create_object` on the battlefield. Set its power, toughness, card_types, and color from params. Use CardId(0) for tokens. Emit TokenCreated events.

8. **life.rs (GainLife)**: Read `LifeAmount` param. Add to controller's life. Emit LifeChanged event with positive amount.

9. **life.rs (LoseLife)**: Read `LifeAmount` param. Subtract from target player's life. Emit LifeChanged event with negative amount. Export as `pub fn resolve_gain` and `pub fn resolve_lose`. Register both "GainLife" and "LoseLife" in the registry.

10. **tap_untap.rs**: For each target Object, set `tapped = true` (Tap) or `tapped = false` (Untap). Emit PermanentTapped/PermanentUntapped events. Export `pub fn resolve_tap` and `pub fn resolve_untap`. Register both.

11. **counters.rs (AddCounter)**: Read `CounterType` and `CounterNum` params (default "1"). For each target Object, increment the counter in obj.counters HashMap. Map CounterType string ("P1P1" -> Plus1Plus1, "M1M1" -> Minus1Minus1, "LOYALTY" -> Loyalty, anything else -> Generic). Emit CounterAdded events.

12. **counters.rs (RemoveCounter)**: Similar to AddCounter but decrements, clamping at 0. Emit CounterRemoved events. Export `pub fn resolve_add` and `pub fn resolve_remove`. Register both.

13. **sacrifice.rs**: For each target Object controlled by ability.controller on battlefield, move to owner's graveyard. Emit PermanentSacrificed events. If no explicit targets, use "Defined$ You" pattern to sacrifice controller's permanents matching a filter (stub the filter for now -- just sacrifice the target).

14. **discard.rs**: Read `NumCards` param (default 1). For controller's hand, move cards to graveyard. If targets specify specific cards, discard those; otherwise discard from end of hand (random selection deferred). Emit Discarded events.

Each handler should emit an `EffectResolved { api_type, source_id }` event after completing.

**Update game/mod.rs**: Add `pub mod effects;` declaration.

**Tests in each handler file**: Unit test with a minimal GameState setup proving the handler mutates state correctly and emits expected events. At minimum test the happy path for each handler.

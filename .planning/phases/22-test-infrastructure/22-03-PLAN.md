**Combat Tests** (`combat.rs`) -- 5+ scenarios. Each with CR doc comment.

1. `/// CR 510.1: Unblocked attacker deals combat damage to defending player`
   - 2/2 attacks, unblocked. Defending player takes 2 damage.
   - Assert: life(P1) == 18.

2. `/// CR 510.1c: Blocked creature and blocker exchange damage`
   - 3/3 attacks, blocked by 2/2. Both take damage. 2/2 dies (lethal damage), 3/3 survives with 2 damage.
   - Assert: Blocker in graveyard, attacker has damage_marked == 2.

3. `/// CR 510.1b: First strike damage resolves before regular damage`
   - 2/2 first strike attacks, blocked by 3/2. First strike kills 3/2 before regular damage.
   - Assert: Blocker in graveyard, attacker takes 0 damage (blocker died before dealing damage).
   - **Snapshot:** `insta::assert_json_snapshot!("combat_first_strike_kills_before_regular", result.snapshot());`

4. `/// CR 510.1c: Double strike deals damage in both steps`
   - 3/3 double strike attacks, blocked by 5/5. First strike step: 3 damage to blocker. Regular step: 3 more damage. Total 6 on 5/5 = lethal.
   - Assert: Blocker in graveyard from 6 total damage.

5. `/// CR 702.2b: Defender can't attack`
   - Creature with defender. Attempt to declare it as attacker.
   - Assert: Engine rejects or excludes from valid_attacker_ids.

6. `/// CR 510.1: Multiple attackers and blockers resolve correctly`
   - Two attackers (3/3 and 2/2). One blocked, one unblocked.
   - Assert: Unblocked attacker deals damage to player, blocked pair exchanges damage.
   - **Snapshot:** `insta::assert_json_snapshot!("combat_multiple_attackers_mixed_blocking", result.snapshot());`

**Keyword Interaction Tests** (`keywords.rs`) -- 5+ scenarios covering keyword combinations:

1. `/// CR 702.2c + CR 702.19b: Deathtouch + trample -- assigns lethal (1) to each blocker, tramples rest`
   - 5/5 deathtouch+trample attacks, blocked by 2/2. Only 1 damage needed to kill (deathtouch), 4 tramples to player.
   - Assert: Blocker in graveyard, defending player loses 4 life.
   - **Snapshot:** `insta::assert_json_snapshot!("keywords_deathtouch_trample_damage_assignment", result.snapshot());`

2. `/// CR 702.15b: Lifelink -- controller gains life equal to damage dealt`
   - 3/3 lifelink attacks unblocked. Controller gains 3 life.
   - Assert: life(P0) == 23.

3. `/// CR 702.15b + CR 510.1b: Lifelink + first strike -- life gained in first strike step`
   - 2/2 lifelink+first strike attacks, blocked by 3/3. First strike kills nothing (2 < 3), but P0 gains 2 life from first strike damage. Then regular damage: 3/3 deals 3 to 2/2, killing it.
   - Assert: P0 gained 2 life, attacker destroyed by blocker.

4. `/// CR 702.9a + CR 702.3a: Flying blocked only by flying or reach`
   - Flying attacker. Blocker without flying/reach can't block.
   - Assert: Blocker not in valid_block_targets, or if forced, attack goes through.

5. `/// CR 702.9a + CR 702.17a: Flying creature blocked by reach creature`
   - Flying attacker, reach blocker. Block is legal.
   - Assert: Block succeeds, combat damage exchanged normally.

6. `/// CR 702.15b + CR 702.19a: Trample + lifelink -- excess damage tramples and heals`
   - 5/5 trample+lifelink attacks, blocked by 2/2. 2 to blocker, 3 tramples to player. Controller gains 5 life (total damage dealt).
   - Assert: P1 loses 3 life, P0 gains 5 life, blocker dies.
   - **Snapshot:** `insta::assert_json_snapshot!("keywords_trample_lifelink_excess", result.snapshot());`

7. `/// CR 702.12a: Vigilance -- attacker doesn't tap`
   - Vigilance creature attacks. It remains untapped.
   - Assert: Object is not tapped after being declared as attacker.

**Layer Tests** (`layers.rs`) -- 5+ scenarios covering CR 613:

1. `/// CR 613.1: Continuous effects applied in layer order`
   - Creature with base 2/2. Lord gives +1/+1 (Layer 7c). Another effect sets P/T to 1/1 (Layer 7b).
   - Layer 7b (set) applies before 7c (modify), so result is 2/2 (set to 1/1, then +1/+1).
   - Assert: Object power == 2, toughness == 2.
   - **Snapshot:** `insta::assert_json_snapshot!("layers_set_then_modify", result.snapshot());`

2. `/// CR 613.4: Layer 7a -- defining P/T`
   - Star creature (*/*) with P/T defined by some characteristic. Verify base P/T is computed correctly.
   - If engine doesn't support *, use a simpler case: creature with P/T set by a static ability.

3. `/// CR 613.4c: Layer 7c -- P/T modifications (+N/+N effects)`
   - Two lord effects stacking: creature gets +1/+1 from lord A and +2/+2 from lord B.
   - Assert: Creature power = base + 3, toughness = base + 3.

4. `/// CR 613.4d: Layer 7d -- P/T counters`
   - Creature with +1/+1 counter. Verify counter modifies P/T in the counter sub-layer.
   - Assert: P/T includes counter boost.

5. `/// CR 613.4e: Layer 7e -- P/T switching`
   - If the engine supports P/T switching effects, test that switching applies in the correct layer order (after all other P/T modifications).
   - Otherwise, test that layer 7c modifications from multiple sources combine correctly (timestamp ordering within same layer).

6. `/// CR 613.7: Timestamp ordering within the same layer`
   - Two effects in the same layer. The one with the later timestamp wins (for conflicting set effects) or they both apply (for additive effects).
   - Create two lords with overlapping affected creatures. Verify both bonuses stack.
   - **Snapshot:** `insta::assert_json_snapshot!("layers_timestamp_ordering", result.snapshot());`

**Note on layer tests:** The existing `layers.rs` inline tests in the engine use `make_creature` + `add_lord_static` + `evaluate_layers()` directly. For integration tests, prefer setting up the scenario via GameScenario and then verifying computed P/T after the engine processes an action (which triggers layer evaluation). If direct layer evaluation is needed, use step-by-step mode and check `game.state()` after layer-triggering actions.

  <action>
Create `crates/engine/src/game/scenario.rs` behind `#[cfg(test)]` gate. This is a single, substantial module implementing the entire test harness API.

**GameScenario struct** (mutable builder):
- `new()` -> Self. Internally holds a GameState via `new_two_player(42)` plus bookkeeping.
- `at_phase(phase: Phase)` -> &mut Self. Sets `state.phase`, `state.waiting_for = WaitingFor::Priority { player: state.active_player }`, `state.priority_player = state.active_player`, `state.turn_number = 2` (avoid summoning sickness issues). Per pitfall #2 from RESEARCH.
- `with_life(player: PlayerId, life: i32)` -> &mut Self.
- `add_creature(player: PlayerId, name: &str, power: i32, toughness: i32)` -> CardBuilder. Creates object via `create_object()` on Battlefield, sets CoreType::Creature, power, toughness, base_power, base_toughness, entered_battlefield_turn = turn_number.saturating_sub(1). Per pitfall #1 and #3 and #4 from RESEARCH.
- `add_vanilla(player: PlayerId, power: i32, toughness: i32)` -> ObjectId. Shorthand, auto-names "N/N Vanilla".
- `add_basic_land(player: PlayerId, color: ManaColor)` -> ObjectId. On Battlefield, CoreType::Land, with mana ability.
- `add_bolt_to_hand(player: PlayerId)` -> ObjectId. Instant in hand with DealDamage{amount:3} effect.
- `add_creature_to_hand(player: PlayerId, name: &str, power: i32, toughness: i32)` -> CardBuilder. In hand zone.
- `build()` -> GameRunner. Consumes self, wraps state.
- `build_and_run(actions: Vec<GameAction>)` -> ScenarioResult. Convenience: build + run all actions.

**CardBuilder<'a> struct:**
Holds `&mut GameScenario` (or `&mut GameState`) + `ObjectId`. Per RESEARCH open question #1, use `&mut GameState` to avoid borrow conflicts. Methods:
- `.flying()`, `.first_strike()`, `.double_strike()`, `.trample()`, `.deathtouch()`, `.lifelink()`, `.vigilance()`, `.haste()`, `.reach()`, `.defender()`, `.menace()`, `.indestructible()`, `.hexproof()`, `.flash()`, `.wither()` -- push corresponding Keyword variant
- `.with_keyword(kw: Keyword)` -- generic fallback
- `.with_ability(effect: Effect)` -- attach AbilityDefinition with effect
- `.with_static(mode: StaticMode)` -- attach StaticDefinition
- `.with_trigger(mode: TriggerMode)` -- attach TriggerDefinition
- `.as_instant()`, `.as_enchantment()`, `.as_sorcery()`, `.as_artifact()` -- change CoreType
- `.with_summoning_sickness()` -- set entered_battlefield_turn = Some(current turn)
- `.with_mana_cost(cost: ManaCost)` -- set mana cost
- All methods return `&mut Self` for chaining.
- Implement `Into<ObjectId>` or `.id()` method to extract ObjectId. Also `Deref<Target = ObjectId>` or similar for ergonomic use in assertions.

**GameRunner struct:**
Wraps `GameState`. Methods:
- `act(action: GameAction)` -> Result<ActionResult, EngineError>. Calls `apply(&mut self.state, action)`.
- `state()` -> &GameState.
- `pass_both_players()` -- passes priority twice (P0 then P1 or current order).
- `resolve_top()` -- passes priority until top of stack resolves.
- `run(actions: Vec<GameAction>)` -> ScenarioResult. Executes all, collects all events.

**ScenarioResult struct:**
Holds final `GameState` + all collected `Vec<GameEvent>`. Query methods:
- `zone(id: ObjectId)` -> Zone
- `life(player: PlayerId)` -> i32
- `battlefield_count(player: PlayerId)` -> usize
- `graveyard_count(player: PlayerId)` -> usize
- `hand_count(player: PlayerId)` -> usize
- `object(id: ObjectId)` -> &GameObject (for detailed assertions)
- `events()` -> &[GameEvent]
- `snapshot()` -> GameSnapshot

**GameSnapshot struct** (Serialize, Deserialize):
Focused projection for insta snapshots. Fields:
- `players`: Vec of `{ life, hand: Vec<String>, graveyard: Vec<String> }` (card names, not IDs -- per pitfall #5)
- `battlefield`: Vec of `{ name, owner, power, toughness, tapped, damage, keywords }` (per player, sorted by name for stability)
- `stack`: Vec of `{ description: String }` (human-readable stack entries)
- `events`: Vec of simplified event descriptions (GameEvent serialized or summarized)
- Use card names and descriptions, NOT raw ObjectIds, to avoid pitfall #5 (ID brittleness).

Register in `crates/engine/src/game/mod.rs`: add `#[cfg(test)] pub mod scenario;` and remove `pub mod test_helpers;`.

**Public constants:** `pub const P0: PlayerId = PlayerId(0); pub const P1: PlayerId = PlayerId(1);`

Write unit tests in inline `#[cfg(test)]` module inside scenario.rs covering all behavior points above.

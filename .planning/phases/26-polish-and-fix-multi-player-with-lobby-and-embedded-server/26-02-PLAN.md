  <name>Task 2: Wire lobby protocol into phase-server message handler</name>
  <files>crates/phase-server/src/main.rs</files>
  <action>
1. Add `LobbyManager` to shared state:
   - Add `SharedLobby = Arc<Mutex<LobbyManager>>` type alias
   - Create lobby instance alongside SessionManager in `main()`
   - Add to router state tuple: `(state, connections, db, lobby)`
   - Add lobby subscriber tracking: `SharedLobbySubscribers = Arc<Mutex<Vec<mpsc::UnboundedSender<ServerMessage>>>>` — each subscribed socket's sender channel

2. Add lobby subscriber tracking per-socket:
   - Add `lobby_subscribed: bool` to `SocketIdentity`
   - Track each socket's tx sender in the lobby subscribers list

3. Handle new ClientMessage variants in `handle_client_message`:
   - `SubscribeLobby`: Add this socket's tx to lobby subscribers list, send current `LobbyUpdate { games }` and `PlayerCount { count }` immediately
   - `UnsubscribeLobby`: Remove this socket's tx from lobby subscribers
   - `CreateGameWithSettings { deck, display_name, public, password, timer_seconds }`:
     - Resolve deck, call `session_manager.create_game_with_settings(resolved, display_name.clone(), timer_seconds)`
     - Register game in LobbyManager with `lobby.register_game(&game_code, display_name, public, password, timer_seconds)`
     - Send `GameCreated` response to the creator
     - If public, broadcast `LobbyGameAdded` to all lobby subscribers
     - Broadcast updated `PlayerCount` to subscribers
   - `JoinGameWithPassword { game_code, deck, display_name, password }`:
     - Check password with `lobby.verify_password(&game_code, password.as_deref())`
     - If password wrong, send `ServerMessage::Error { message: "Wrong password" }`
     - If password check requires a password but none given, send `PasswordRequired { game_code }`
     - On success: call `session_manager.join_game_with_name(&game_code, resolved, display_name)`
     - Unregister game from lobby: `lobby.unregister_game(&game_code)`
     - Send `GameStarted` to both players (same pattern as existing JoinGame handler)
     - Broadcast `LobbyGameRemoved { game_code }` to lobby subscribers
     - Broadcast updated `PlayerCount`
   - `Concede`:
     - Look up game_code and player from identity
     - Opponent wins: broadcast `ServerMessage::GameOver { winner: Some(opponent), reason: "Opponent conceded" }` to both players
     - Also broadcast `ServerMessage::Conceded { player }` for UI animation before game over
     - Clean up session
   - `Emote { emote }`:
     - Look up game_code and player from identity
     - Forward `ServerMessage::Emote { from_player: player, emote }` to the opponent only

4. Extend the background expiry task:
   - Also check `lobby.check_expired(300)` (5 minute timeout for waiting games)
   - For each expired game code, remove from session manager, broadcast `LobbyGameRemoved` to subscribers
   - Continue checking reconnect expiry as before

5. On socket disconnect:
   - If `lobby_subscribed`, remove from subscribers list
   - Decrement and broadcast updated `PlayerCount`

6. Player count tracking: count total connected sockets (increment on new socket open, decrement on close). Broadcast `PlayerCount` on change to all lobby subscribers.

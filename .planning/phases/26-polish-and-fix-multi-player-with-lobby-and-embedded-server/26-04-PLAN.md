<objective>
Port Alchemy's WebRTC P2P networking code and implement P2PHostAdapter/P2PGuestAdapter for browser-based multiplayer.

Purpose: Browser/PWA users cannot run the phase-server, so they need P2P hosting. The host runs the WASM engine locally and relays filtered state to the guest via WebRTC DataChannel. This reuses Alchemy's proven PeerJS implementation.

Output: network/ module (connection.ts, peer.ts, protocol.ts) ported from Alchemy, plus P2PHostAdapter and P2PGuestAdapter implementing EngineAdapter, wired into GameProvider. Test scaffolds for P2P message handling.

1. Create `client/src/adapter/p2p-adapter.ts` with two classes:

   **P2PHostAdapter** (implements EngineAdapter):
   - Constructor takes: `deckData: unknown` (player deck payload), `session: PeerSession`
   - Has internal `WasmAdapter` instance for local engine execution
   - `initialize()`: Create WasmAdapter, call `wasm.initialize()`, wait for guest's `guest_deck` message, then call `wasm.initializeGame(fullPayload)` where fullPayload combines host + guest deck
   - `initializeGame()`: After receiving guest deck and initializing WASM, send `game_setup` message to guest with the initial filtered state
   - `submitAction(action)`:
     - Apply action locally via `wasm.submitAction(action)`
     - Get full state via `wasm.getState()`
     - Filter state for guest: `filterStateForGuest(state)` — hide host's hand/library from guest view (simple TS function: replace opponent's hand with empty, opponent's library with count-only)
     - Send `state_update` to guest via session
     - Return events
   - `getState()`: Delegate to wasm
   - `getLegalActions()`: Delegate to wasm
   - `getAiAction()`: Return null (no AI in P2P)
   - `restoreState()`: Throw (no undo in P2P)
   - `dispose()`: Close session, dispose wasm
   - Listen for `action` messages from guest: validate via `wasm.getLegalActions()`, if legal apply via `wasm.submitAction()`, send `state_update` back. If illegal, send `action_rejected`.
   - Listen for `concede` from guest: end game, host wins
   - Emit events for UI (use same WsAdapterEvent pattern or a new P2PAdapterEvent type)

   **P2PGuestAdapter** (implements EngineAdapter):
   - Constructor takes: `deckData: unknown` (guest deck), `session: PeerSession`
   - Stores `gameState: GameState | null`
   - `initialize()`: Send `guest_deck` message with deck data. Wait for `game_setup` message. Store initial state.
   - `initializeGame()`: Return events from game_setup
   - `submitAction(action)`: Send `action` message to host. Return promise that resolves when `state_update` arrives. (Same pending resolve pattern as WebSocketAdapter.)
   - `getState()`: Return stored gameState
   - `getLegalActions()`: Return empty (host validates)
   - `getAiAction()`: Return null
   - `restoreState()`: Throw
   - `dispose()`: Close session
   - Listen for `state_update`: update gameState, resolve pending. Also emit `stateChanged` for unsolicited updates (same Bug C pattern).

   - Keep everything else (battlefield, graveyard, exile are public zones)

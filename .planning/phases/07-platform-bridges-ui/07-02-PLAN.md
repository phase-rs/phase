<task type="auto">
  <name>Task 1: Scryfall API client, IndexedDB cache service, and useCardImage hook</name>
  <files>client/src/services/scryfall.ts, client/src/services/imageCache.ts, client/src/hooks/useCardImage.ts</files>
  <action>
1. Create `client/src/services/imageCache.ts`:
   - Use `idb-keyval` (get, set) for IndexedDB access.
   - `getCachedImage(cardName: string, size: 'small' | 'normal' | 'large'): Promise<string | null>` -- checks IndexedDB for key `scryfall:${cardName}:${size}`, if found creates object URL from Blob and returns it.
   - `cacheImage(cardName: string, size: string, blob: Blob): Promise<void>` -- stores Blob in IndexedDB.
   - `revokeImageUrl(url: string): void` -- revokes object URL to prevent memory leaks.

2. Create `client/src/services/scryfall.ts`:
   - `SCRYFALL_DELAY_MS = 75` constant.
   - Internal fetch queue to serialize requests with delay between each.
   - `fetchCardImage(cardName: string, size: 'normal' | 'large' | 'small' = 'normal'): Promise<Blob>`:
     - Check cache first via imageCache.getCachedImage.
     - On cache miss: fetch `https://api.scryfall.com/cards/named?exact=${encodeURIComponent(cardName)}`.
     - Parse response JSON. Get image URL from `image_uris[size]` or `card_faces[0].image_uris[size]` for DFCs.
     - Fetch image blob, cache it, return blob.
   - `prefetchDeckImages(cardNames: string[]): Promise<void>`:
     - Deduplicate names, fetch each with SCRYFALL_DELAY_MS between requests.
     - Pre-fetch 'normal' size for all cards.
   - `fetchCardImageUrl(cardName: string, faceIndex: number, size: string): Promise<string>`:
     - For DFCs, use `card_faces[faceIndex].image_uris[size]`.
     - Returns the image URL string (for cases where we need the URL, not blob).

3. Create `client/src/hooks/useCardImage.ts`:
   - `useCardImage(cardName: string, options?: { size?: 'small' | 'normal' | 'large', faceIndex?: number }): { src: string | null, isLoading: boolean }`.
   - On mount/cardName change: check cache, if miss fetch from Scryfall.
   - Returns object URL string for the image blob, or null while loading.
   - Cleanup: revoke object URLs on unmount via revokeImageUrl.

4. Convert the imageCache test stubs from 07-00 into real tests: mock idb-keyval's get/set, verify getCachedImage returns null on miss, verify cacheImage stores and retrieves blobs.

     - F: Toggle Full Control (uiStore.toggleFullControl).

   - Before each dispatch, if the action is an "unrevealed-information action" (PassPriority, DeclareAttackers, DeclareBlockers, ActivateAbility — NOT CardDrawn which reveals info), push a serialized copy of current gameState to stateHistory array (max 5 entries).
   - `undo()` action: if stateHistory is non-empty, pop last state, set as current gameState. Clear the events array. Mark canUndo = false until next action.
   - `canUndo` selector: true if stateHistory is non-empty.

1. Create `client/src/adapter/tauri-adapter.ts`:
   - Implements EngineAdapter interface.
   - Uses dynamic import of `@tauri-apps/api/core` (invoke function) to avoid bundling Tauri API in web builds.
   - `initialize()`: calls `invoke('initialize_game')`.
   - `submitAction(action)`: calls `invoke('submit_action', { action })`, returns GameEvent[].
   - `getState()`: calls `invoke('get_game_state')`, returns GameState.
   - `dispose()`: calls `invoke('dispose_game')`, catch and ignore errors.
   - Type-safe: invoke returns typed results matching serde serialization.
   - Note: The actual Tauri Rust commands (in src-tauri/) are NOT created here — just the TypeScript adapter. Tauri app scaffolding (Cargo.toml, tauri.conf.json, src-tauri/main.rs) would need `@tauri-apps/cli init` which is separate infrastructure work. This plan creates the adapter that will work once the Tauri backend exists.

2. Create `client/src/adapter/index.ts`:
   - `createAdapter(): EngineAdapter`:
     - Detect platform: check for `window.__TAURI_INTERNALS__` (Tauri v2 detection).
     - If Tauri detected: dynamically import and return TauriAdapter.
     - Otherwise: return WasmAdapter.
   - Re-export all adapter types from types.ts.
   - Re-export WasmAdapter for direct use.

3. Create `client/public/manifest.json`:
   - PWA manifest with: name "Forge.ts", short_name "Forge.ts", start_url "/", display "standalone", background_color "#030712" (gray-950), theme_color "#030712".
   - Icons: placeholder 192x192 and 512x512 (generate simple icon or use placeholder).

4. Update `client/vite.config.ts`:
   - Add `vite-plugin-pwa` (install: `pnpm add -D vite-plugin-pwa`).
   - Configure VitePWA plugin with: registerType 'autoUpdate', workbox runtime caching for Scryfall CDN images (CacheFirst strategy), manifest from public/manifest.json.
   - This enables browser "Install App" prompt and offline image caching via service worker.

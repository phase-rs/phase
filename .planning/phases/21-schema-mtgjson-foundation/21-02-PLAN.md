<objective>
Refactor all ~13 consumer files to use typed ability structs instead of Vec<String>. This is the core type migration that makes the engine consume typed definitions end-to-end.

Purpose: Eliminate runtime string parsing of abilities -- the engine receives typed data from parse time through to effect resolution. This is the locked user decision: "a schema the engine can't consume is academic."
Output: All ability-related fields across CardFace, GameObject, BackFaceData, FaceDownData use typed definition vectors. Effect resolution uses pattern matching on Effect enum.
</objective>

<objective>
Refactor all remaining string-based dispatch in the engine to typed pattern matching on enums.

Purpose: This eliminates the dependency on compat bridge methods (api_type(), mode_str(), event_str(), params()) from ungated production code, which is the prerequisite for feature-gating them behind `forge-compat` in Plan 02. The effect handler registry was already converted to typed dispatch in Phase 21-04; this plan completes the migration for the remaining ~13 dispatch sites (static abilities, replacement effects, combat, layers, coverage, casting, mana abilities, planeswalker) plus the 40+ `api_type()` calls in effect handlers used for GameEvent::EffectResolved strings.

Output: All engine dispatch uses typed enums. No compat bridge method is called from ungated production code. All 800+ tests pass.
</objective>

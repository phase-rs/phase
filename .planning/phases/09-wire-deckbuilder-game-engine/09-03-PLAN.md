**1. Update EngineAdapter interface** (`client/src/adapter/types.ts`):
- Add `initializeGame(deckData?: unknown): Promise<GameEvent[]> | GameEvent[];` to the `EngineAdapter` interface. This makes deck data passing a first-class part of the adapter contract.

**2. Update WasmAdapter** (`client/src/adapter/wasm-adapter.ts`):
- Modify the existing `initializeGame` method: it already exists and calls `initialize_game(deckData ?? null)`. No changes needed to the method itself, but make sure it conforms to the updated interface.
- In the `initialize()` method, do NOT call initializeGame -- that's done separately by the store.

**3. Update gameStore.initGame** (`client/src/stores/gameStore.ts`):
- Change signature: `initGame: async (adapter: EngineAdapter, deckData?: unknown) => { ... }`
- After `await adapter.initialize()`, call `adapter.initializeGame(deckData)` to pass deck data to the engine
- Then call `await adapter.getState()` to get the initial state with populated libraries

**4. Update GamePage** (`client/src/pages/GamePage.tsx`):
- Import `ACTIVE_DECK_KEY` and `STORAGE_KEY_PREFIX` from `../constants/storage`
- Replace `loadDeckFromSession()` function: read active deck name from `localStorage.getItem(ACTIVE_DECK_KEY)`, then load deck data from `localStorage.getItem(STORAGE_KEY_PREFIX + activeDeckName)`
- Add redirect guard: if no active deck found in localStorage, call `navigate("/")` and return early from the useEffect
- For the deck data passed to WASM: load the pre-computed card data JSON file (from `/card-data.json` -- generated by `card_data_export` CLI). Use `fetch('/card-data.json')` to load it. For each card name in the deck, look up the CardFace definition and build the `DeckPayload` object:
  ```typescript
  interface DeckPayload {
    player_deck: Array<{ card: CardFace; count: number }>;
    opponent_deck: Array<{ card: CardFace; count: number }>;
  }
  ```
- If card-data.json is not available (404), fall back to passing null to initializeGame (empty game). Log a warning.
- For the opponent's deck: use the same deck as the player (mirror match) for now. This is the simplest viable approach and can be refined later.
- Pass the constructed DeckPayload to `initGame(adapter, deckPayload)` in the useEffect
- For online mode, the deck loading remains as-is (server handles decks)

**5. Remove the old `loadDeckFromSession`** function entirely. The new flow reads from localStorage using shared constants.

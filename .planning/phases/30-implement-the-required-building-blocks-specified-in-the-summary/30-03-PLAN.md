**Adventure Casting Choice (casting.rs):**

1. In `handle_cast_spell()`: after finding the card object, before timing validation, check if the card has `CardLayout::Adventure`. If so, and the card is being cast from hand (Zone::Hand), the player needs to choose which face to cast. Return `WaitingFor::AdventureCastChoice { player, object_id, card_id }` to prompt the player.

2. Add `WaitingFor::AdventureCastChoice` variant to `WaitingFor` enum in `types/game_state.rs`:
   ```rust
   /// CR 715.3a: Player chooses creature face vs Adventure half.
   AdventureCastChoice {
       player: PlayerId,
       object_id: ObjectId,
       card_id: CardId,
   },
   ```

3. Add `GameAction::ChooseAdventureFace { creature: bool }` variant to `GameAction` enum. `creature: true` means cast as creature, `false` means cast as Adventure spell.

4. In `engine.rs::apply()`: add match arm for `(WaitingFor::AdventureCastChoice { player, object_id, card_id }, GameAction::ChooseAdventureFace { creature })`:
   - If `creature`: continue normal casting flow with the creature face's characteristics
   - If Adventure (`creature == false`):
     - Swap the object's active characteristics to the Adventure face (name, mana cost, types, abilities)
     - Mark the stack entry with `cast_as_adventure: true` (see below)
     - Continue casting with Adventure face mana cost and abilities

**Stack Entry Adventure Tracking (game_state.rs + stack.rs):**

5. Add `cast_as_adventure: bool` field to `StackEntryKind::Spell`:
   ```rust
   Spell {
       card_id: CardId,
       ability: ResolvedAbility,
       #[serde(default)]
       cast_as_adventure: bool,
   },
   ```

6. In `resolve_top()`: after the spell resolves successfully and BEFORE moving to graveyard, check if `cast_as_adventure` is true:
   ```rust
   // CR 715.4: Adventure spell resolves → exile with casting permission
   if cast_as_adventure {
       // Move to exile instead of graveyard
       zones::move_to_zone(state, card_id, Zone::Exile, events);
       // Add casting permission for creature face
       if let Some(obj) = state.objects.get_mut(&entry.source_id) {
           obj.casting_permissions.push(CastingPermission::AdventureCreature);
       }
   } else {
       // Normal: move to graveyard
       zones::move_to_zone(state, card_id, Zone::Graveyard, events);
   }
   ```

7. **Countered Adventure spells:** When a spell is countered (fizzled), it already goes to graveyard via the existing fizzle path. No change needed — CR 715.4 says only RESOLUTION triggers exile. Verify this is the case by tracing the fizzle code path.

**Casting from Exile (casting.rs):**

8. In `handle_cast_spell()` (or a new `can_cast_from_zone()` check): when a player attempts to cast a card from exile, check if the card has `CastingPermission::AdventureCreature` in `casting_permissions`. If so, allow casting as the creature face.

9. When a card with `AdventureCreature` permission is cast from exile:
   - Use the creature face characteristics (the primary face)
   - The `CastingPermission::AdventureCreature` is consumed — remove it from the object's permissions. Or, since the object moves from exile to stack on casting, the permission becomes irrelevant (the object is no longer in exile). If the creature is returned to exile later, it should NOT still have the permission. Ensure the permission is cleared when the object leaves exile via zone change.

**WaitingFor in casting.rs:**

10. In the existing flow where spells in hand are validated for casting, also scan exile zone objects for cards with `AdventureCreature` permission and include them as castable options during priority. This integrates with the legal actions system.

Create `client/src/audio/sfxMap.ts`:
- Export `SFX_MAP: Record<string, string>` mapping 15 GameEvent type strings to Forge SFX filenames (without extension):
  - DamageDealt -> "destroy", LifeChanged -> "life_loss", SpellCast -> "instant", CreatureDestroyed -> "destroy", AttackersDeclared -> "creature", BlockersDeclared -> "block", LandPlayed -> "green_land", CardDrawn -> "draw", SpellCountered -> "sorcery", TokenCreated -> "token", GameStarted -> "shuffle", GameOver -> "win_duel", PermanentSacrificed -> "destroy", CounterAdded -> "add_counter", AbilityActivated -> "enchant"
- Export `MUSIC_TRACKS: string[]` — array of 4 filenames (without extension): ["Dangerous", "Failing Defense", "Hitman", "Prelude and Action"]

Create `client/src/audio/AudioManager.ts`:
- Plain TypeScript class (NOT a React component). Module-level singleton `audioManager` exported.
- Export standalone function `initAudioOnInteraction()` that attaches one-shot click/touchstart/keydown listeners to `document`. On first interaction: calls `audioManager.warmUp()`, then `audioManager.preloadSfx()`. Removes all 3 listeners after first fire.
- AudioManager class:
  - Private fields: `ctx: AudioContext | null`, `sfxBuffers: Map<string, AudioBuffer>`, `sfxGain: GainNode | null`, `musicGain: GainNode | null`, `currentAudio: HTMLAudioElement | null`, `currentSource: MediaElementAudioSourceNode | null`, `trackOrder: string[]`, `trackIndex: number`, `isWarmedUp: boolean`
  - `warmUp()`: Create AudioContext if not exists. Create sfxGain and musicGain nodes connected to destination. Apply saved preferences from `usePreferencesStore.getState()` (sfxVolume, musicVolume, sfxMuted, musicMuted, masterMuted). If masterMuted, set both gains to 0. Set isWarmedUp = true.
  - `preloadSfx()`: If no ctx, return. Get unique filenames from SFX_MAP values. For each, fetch `/audio/sfx/${filename}.m4a`, decode to AudioBuffer, store in sfxBuffers map. Use Promise.all for parallel loading. console.warn on individual failures (don't break the batch).
  - `playSfx(eventType: string, volume?: number)`: Look up SFX_MAP[eventType]. If no mapping or no buffer or muted or no ctx, return silently. Create BufferSourceNode, connect to sfxGain (with intermediate GainNode if volume !== 1.0 for consolidation boost). Start immediately.
  - `playSfxForStep(effects: StepEffect[])`: Group effects by type, count occurrences. For each unique type with an SFX mapping, call playSfx with volume boost: `count > 1 ? Math.min(1.0 + count * 0.15, 1.5) : 1.0`. This implements the consolidation requirement.
  - `startMusic()`: If no ctx/musicGain or music muted+master muted, return. Shuffle MUSIC_TRACKS using Fisher-Yates into trackOrder. Set trackIndex = 0. Call playTrack().
  - `playTrack()`: Create new HTMLAudioElement for `/audio/music/${trackOrder[trackIndex]}.m4a`. Create MediaElementSourceNode, connect to musicGain. Play. On 'ended', advance trackIndex (wrap + re-shuffle if needed), crossfade to next.
  - `crossfadeTo(nextIndex: number, duration = 2.5)`: Using musicGain, ramp current gain to 0 over duration. After timeout, pause old audio, call playTrack for nextIndex with fade-in ramp from 0 to target volume. Use `cancelScheduledValues` + `setValueAtTime` before any ramp to avoid Pitfall 3 (gain scheduling conflicts).
  - `stopMusic(fadeOut = 2.0)`: Fade out music gain over fadeOut seconds, then pause audio element. For GameOver.
  - `updateVolumes()`: Read current preferences via getState(). Cancel any scheduled gains. Set sfxGain.value based on sfxMuted/masterMuted/sfxVolume. Set musicGain.value based on musicMuted/masterMuted/musicVolume. This is called by the preferences store subscription (wired in Plan 03).
  - `dispose()`: Stop music, close AudioContext. For cleanup.
- Subscribe AudioManager to preferencesStore changes for real-time volume updates: `usePreferencesStore.subscribe((state) => audioManager.updateVolumes())` at module level.

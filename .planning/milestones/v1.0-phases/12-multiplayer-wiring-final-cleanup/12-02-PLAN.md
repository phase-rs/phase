<task type="auto">
  <name>Task 1: Add reconnect retry logic and new event types to WebSocketAdapter</name>
  <files>client/src/adapter/ws-adapter.ts</files>
  <action>
    **Extend WsAdapterEvent union** with three new variants:
    ```typescript
    | { type: "reconnecting"; attempt: number; maxAttempts: number }
    | { type: "reconnected" }
    | { type: "reconnectFailed" }
    ```

    **Add private state to WebSocketAdapter:**
    - `private reconnectAttempt = 0;`
    - `private readonly maxReconnectAttempts = 3;`
    - `private reconnectTimer: ReturnType<typeof setTimeout> | null = null;`
    - `private disposed = false;`

    **Add `private attemptReconnect()` method:**
    1. If `this.disposed`, return immediately (prevents reconnect after intentional navigation)
    2. If `this.reconnectAttempt >= this.maxReconnectAttempts`, emit `{ type: "reconnectFailed" }` and return
    3. Increment `this.reconnectAttempt`
    4. Calculate delay: `Math.pow(2, this.reconnectAttempt - 1) * 1000` (1s, 2s, 4s)
    5. Emit `{ type: "reconnecting", attempt: this.reconnectAttempt, maxAttempts: this.maxReconnectAttempts }`
    6. Set `this.reconnectTimer = setTimeout(() => { this.tryReconnect(); }, delay)`

    **Modify `ws.onclose` handler** in `initialize()`:
    - After the existing initReject check (pre-game-start close), add: if game has started (initResolve is null and gameState exists), call `this.attemptReconnect()`

    **Modify `tryReconnect()`:**
    - After successful reconnect (GameStarted message received), emit `{ type: "reconnected" }` and reset `this.reconnectAttempt = 0`
    - Wire the new WebSocket's `onclose` to also call `attemptReconnect()` (reconnect session can also drop)
    - Wire the new WebSocket's `onerror` similarly

    **Modify `dispose()`:**
    - Set `this.disposed = true` BEFORE `this.ws.close()` so onclose doesn't trigger reconnect
    - Clear `this.reconnectTimer` if set: `if (this.reconnectTimer) { clearTimeout(this.reconnectTimer); this.reconnectTimer = null; }`

    **Critical pitfall to handle:** The existing `ws.onclose` in `initialize()` fires when socket closes pre-game (rejects initPromise). Only trigger reconnect if the game has actually started (i.e., `this.gameState !== null` and `this.initResolve === null`).
  </action>

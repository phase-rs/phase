   - `pub struct DeckData { pub main_deck: Vec<String>, pub sideboard: Vec<String> }` (card names)
   - `pub enum ClientMessage` (serde tagged): CreateGame { deck }, JoinGame { game_code, deck }, Action { action: GameAction }, Reconnect { game_code, player_token }
   - `pub enum ServerMessage` (serde tagged): GameCreated { game_code, player_token }, GameStarted { state: GameState, your_player: PlayerId }, StateUpdate { state: GameState, events: Vec<GameEvent> }, ActionRejected { reason }, OpponentDisconnected { grace_seconds: u32 }, OpponentReconnected, GameOver { winner: Option<PlayerId>, reason }, Error { message }
   - All types derive Serialize + Deserialize

3. Create `crates/server-core/src/filter.rs`:
   - `pub fn filter_state_for_player(state: &GameState, player: PlayerId) -> GameState`:
     - Clone the state
     - For the opponent: iterate hand objects, set face_down = true, clear name/abilities/keywords (replace name with "Hidden Card")
     - For both players: don't expose library contents (libraries are already order-sensitive — just ensure the object details in library are hidden for opponent)
     - Allowlist approach: explicitly include safe fields rather than trying to strip sensitive ones
   - Write unit tests: filtered state has opponent hand size preserved but card identities hidden; own hand is fully visible; library contents hidden

4. Create `crates/server-core/src/reconnect.rs`:
   - `pub struct ReconnectManager { grace_period: Duration, disconnected: HashMap<String, DisconnectInfo> }` where DisconnectInfo has player_id, disconnect_time, game_code
   - `pub fn record_disconnect(&mut self, game_code: &str, player: PlayerId)` — record disconnect time
   - `pub fn attempt_reconnect(&mut self, game_code: &str, player_token: &str) -> ReconnectResult` — check if within grace period, return Ok(game_code) or Err(expired)
   - `pub fn check_expired(&mut self) -> Vec<String>` — return game codes with expired grace periods (for forfeit)
   - Default grace period: 120 seconds (2 minutes)
   - Write unit tests: reconnect within grace period succeeds, reconnect after expiry fails

5. Create `crates/server-core/src/session.rs`:
   - `pub struct GameSession { game_code: String, state: GameState, player_tokens: [String; 2], connected: [bool; 2] }`
   - `pub struct SessionManager { sessions: HashMap<String, GameSession>, reconnect: ReconnectManager }`
   - `impl SessionManager`:
     - `pub fn create_game(&mut self, deck: DeckData) -> (String, String)` — generate 6-char alphanumeric game_code and player_token, create GameSession with player 0's deck, return (game_code, player_token)
     - `pub fn join_game(&mut self, game_code: &str, deck: DeckData) -> Result<(String, GameState), String>` — find session, add player 1's deck, initialize game via engine start_game, return (player_token, initial_filtered_state)
     - `pub fn handle_action(&mut self, game_code: &str, player_token: &str, action: GameAction) -> Result<(GameState, GameState, Vec<GameEvent>), String>` — validate token matches active player, validate action is legal via get_legal_actions, apply action, return (filtered_state_p0, filtered_state_p1, events)
     - `pub fn handle_disconnect(&mut self, game_code: &str, player: PlayerId)` — mark disconnected, start grace period
     - `pub fn handle_reconnect(&mut self, game_code: &str, player_token: &str) -> Result<GameState, String>` — verify token, return filtered state snapshot
   - Generate game_code as random 6-char uppercase alphanumeric
   - Generate player_token as random 32-char hex string
   - Write unit tests: create -> join -> action cycle works; action from wrong player rejected; illegal action rejected

1. Create `crates/forge-server/Cargo.toml`:
   - Dependencies: `server-core = { path = "../server-core" }`, `engine = { path = "../engine" }`, `axum = { version = "0.8", features = ["ws"] }`, `tokio = { version = "1", features = ["full"] }`, `tower-http = { version = "0.6", features = ["cors"] }`, `serde = { workspace = true }`, `serde_json = "1"`
   - `[[bin]] name = "forge-server" path = "src/main.rs"`

2. Create `crates/forge-server/src/main.rs`:
   - Shared state: `Arc<Mutex<SessionManager>>` (tokio Mutex for async context)
   - Also maintain a mapping of game_code -> connected WebSocket senders for broadcasting: `Arc<Mutex<HashMap<String, HashMap<PlayerId, mpsc::UnboundedSender<ServerMessage>>>>>`
   - Routes:
     - `GET /ws` — WebSocket upgrade handler
     - `GET /health` — returns "ok" (for monitoring)
     - `GET /games` — list open games waiting for opponents (for lobby browser)
   - `async fn handle_socket(socket: WebSocket, state: SharedState, connections: SharedConnections)`:
     - Split socket into sender/receiver
     - Read messages in a loop, parse as ClientMessage:
       - `CreateGame`: call session_manager.create_game, send GameCreated response, register sender in connections map
       - `JoinGame`: call session_manager.join_game, send GameStarted to both players (each gets their filtered state), register sender
       - `Action`: call session_manager.handle_action, send StateUpdate to both players (each filtered), or ActionRejected on error
       - `Reconnect`: call session_manager.handle_reconnect, send full filtered state, notify opponent via OpponentReconnected
     - On socket close: call session_manager.handle_disconnect, send OpponentDisconnected to other player
   - CORS middleware via tower-http (allow all origins for dev)
   - Server config: bind to `0.0.0.0:8080` (configurable via PORT env var)
   - Spawn a background task that periodically checks for expired reconnect grace periods (every 10s) and sends GameOver(forfeit) to remaining players
   - Log startup message with port

3. Verify the full workspace compiles: `cargo build --workspace`

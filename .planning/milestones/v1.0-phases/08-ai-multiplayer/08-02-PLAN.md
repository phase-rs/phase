  <files>crates/forge-ai/src/search.rs, crates/forge-ai/src/lib.rs, crates/engine-wasm/src/lib.rs, crates/engine-wasm/Cargo.toml</files>
  <action>
1. Create `crates/forge-ai/src/search.rs`:
   - `struct SearchBudget { max_nodes: u32, nodes_evaluated: u32 }` with `fn exhausted(&self) -> bool` and `fn tick(&mut self)`
   - `struct ScoredAction { action: GameAction, score: f64 }`
   - `pub fn choose_action(state: &GameState, ai_player: PlayerId, config: &AiConfig, rng: &mut impl Rng) -> Option<GameAction>`:
     - Get legal actions via get_legal_actions
     - If 0 or 1 legal action, return immediately (no search needed)
     - If search not enabled (VeryEasy/Easy), use heuristic-only: score each action with should_play_now for Priority, choose_attackers for DeclareAttackers, choose_blockers for DeclareBlockers, then softmax select
     - If search enabled (Medium+), run alpha-beta with iterative deepening
     - For DeclareAttackers: use choose_attackers directly (combinatorial explosion too high for tree search)
     - For DeclareBlockers: use choose_blockers directly
     - Apply temperature via softmax_select
   - `fn search_value(state: &GameState, ai_player: PlayerId, depth: u32, alpha: f64, beta: f64, config: &AiConfig, budget: &mut SearchBudget) -> f64`:
     - Base cases: depth 0 or budget exhausted or GameOver -> evaluate_state
     - Get legal actions, for each: clone state, apply action, recurse with swapped min/max
     - Alpha-beta pruning: skip branches when alpha >= beta
   - `fn softmax_select(scored: &[ScoredAction], temperature: f64, rng: &mut impl Rng) -> Option<GameAction>`:
     - Subtract max score for numerical stability
     - Compute exp(score / temperature) for each action
     - Normalize to probabilities, sample weighted random
     - Temperature near 0 -> deterministic best move; high temperature -> more random
   - Write tests: search at depth 2 prefers winning move over neutral; softmax with temperature 0.01 picks highest score; budget limits stop search

2. Update `crates/forge-ai/src/lib.rs`: export `pub use search::choose_action;`

3. Add forge-ai dependency to `crates/engine-wasm/Cargo.toml`: `forge-ai = { path = "../forge-ai" }`

4. Add WASM binding in `crates/engine-wasm/src/lib.rs`:
   - `#[wasm_bindgen] pub fn get_ai_action(difficulty: &str, platform: &str) -> Result<JsValue, JsValue>`:
     - Parse difficulty string to AiDifficulty enum
     - Parse platform string to Platform enum (always "Wasm" when called from WASM)
     - Create config via create_config(difficulty, platform)
     - Get current game state from thread-local
     - Create an RNG (use rand::thread_rng or seeded from game state)
     - Call choose_action(state, ai_player_id, config, rng)
     - Serialize chosen GameAction to JsValue and return
   - The ai_player is determined by checking which player WaitingFor targets (the AI is always player 1, human is player 0)

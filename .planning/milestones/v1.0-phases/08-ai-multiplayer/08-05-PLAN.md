<objective>
Close the verification gap for Truth 5: "A card coverage dashboard shows which cards and effects are supported vs. missing." The existing dashboard shows handler lists but lacks per-card Standard coverage analysis.

Purpose: The Rust `coverage.rs` already implements `analyze_standard_coverage()` which produces per-card results with missing handler details. The WASM binding was skipped because `CardDatabase` requires filesystem access unavailable in browsers. This plan uses a build-time pre-computation approach: a small Rust binary generates coverage JSON, which the dashboard loads as a static asset.

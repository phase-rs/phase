Fix the undo/WASM state desync bug: after pressing Z to undo, the WASM engine's thread-local GameState must be restored to match the client-side reverted state, so subsequent actions operate on the correct state.

Purpose: Currently undo only reverts client-side state while the WASM engine retains the latest state, causing a desync on the next action.
Output: Working undo that keeps client and WASM engine in sync.
</objective>

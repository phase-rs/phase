<task type="auto" tdd="true">
  <name>Task 2: Implement ability string parser for A:/T:/S:/R: lines</name>
  <files>

  <behavior>
    AbilityDefinition parsing (A: lines):
    - "SP$ DealDamage | ValidTgts$ Any | NumDmg$ 3" → AbilityDefinition { kind: Spell, api_type: "DealDamage", params: {"ValidTgts": "Any", "NumDmg": "3"} }
    - "AB$ Draw | Cost$ T | NumCards$ 1" → AbilityDefinition { kind: Activated, api_type: "Draw", params: {"Cost": "T", "NumCards": "1"} }
    - "DB$ ChangeZone | Origin$ Battlefield" → AbilityDefinition { kind: Database, api_type: "ChangeZone", params: {"Origin": "Battlefield"} }

    TriggerDefinition parsing (T: lines):
    - "Mode$ ChangesZone | Origin$ Any | Destination$ Battlefield | Execute$ TrigDraw" → TriggerDefinition { mode: "ChangesZone", params: {"Origin": "Any", "Destination": "Battlefield", "Execute": "TrigDraw"} }

    StaticDefinition parsing (S: lines):
    - "Mode$ Continuous | Affected$ Card.Self | AddPower$ 2" → StaticDefinition { mode: "Continuous", params: {"Affected": "Card.Self", "AddPower": "2"} }

    ReplacementDefinition parsing (R: lines):
    - "Event$ DamageDone | ActiveZones$ Battlefield | ValidSource$ Card.Self" → ReplacementDefinition { event: "DamageDone", params: {"ActiveZones": "Battlefield", "ValidSource": "Card.Self"} }

    Error case:
    - "NoKind$ Value | Foo$ Bar" → Err(MissingAbilityKind) for ability parse
  </behavior>
  <action>
    1. Create `types/ability.rs`:
       - `AbilityKind` enum: Spell, Activated, Database — all derive Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize
       - `AbilityDefinition` struct: kind (AbilityKind), api_type (String), params (HashMap<String, String>) — derive Debug, Clone, Serialize, Deserialize
       - `TriggerDefinition` struct: mode (String), params (HashMap<String, String>)
       - `StaticDefinition` struct: mode (String), params (HashMap<String, String>)
       - `ReplacementDefinition` struct: event (String), params (HashMap<String, String>)

    2. Create `parser/ability.rs`:
       - `pub fn parse_ability(raw: &str) -> Result<AbilityDefinition, ParseError>`:
         Split on `|`, trim each segment, split on `$` to get key/value pairs.
         SP$/AB$/DB$ set the kind and api_type. All other Key$Value go into params HashMap.
         Error if no SP$/AB$/DB$ found (MissingAbilityKind).
       - `pub fn parse_trigger(raw: &str) -> Result<TriggerDefinition, ParseError>`:
         Same pipe-split approach but `Mode$` is the discriminator. Extract mode value, rest into params.
       - `pub fn parse_static(raw: &str) -> Result<StaticDefinition, ParseError>`:
         Same pipe-split, `Mode$` is discriminator.
       - `pub fn parse_replacement(raw: &str) -> Result<ReplacementDefinition, ParseError>`:
         Same pipe-split, `Event$` is discriminator.
       - Factor common pipe-splitting into a shared helper `fn parse_params(raw: &str) -> HashMap<String, String>` if the pattern is identical.

    3. Update `types/mod.rs` re-exports to include ability types.
  </action>

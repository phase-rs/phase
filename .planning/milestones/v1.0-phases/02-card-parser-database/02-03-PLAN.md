  <name>Task 1: Implement CardDatabase with directory loading and name indexing</name>
  <files>

  <behavior>
    Loading:
    - load(Path) recursively finds all .txt files under the given directory
    - Each .txt file is read and parsed via parse_card_file
    - Parse failures are collected as warnings (not hard errors) — lenient loading matches lenient parsing
    - Returns CardDatabase with all successfully parsed cards indexed

    Lookup by name:
    - get_by_name("Lightning Bolt") → Some(&CardRules)
    - get_by_name("lightning bolt") → Some(&CardRules) (case-insensitive)
    - get_by_name("LIGHTNING BOLT") → Some(&CardRules) (case-insensitive)
    - get_by_name("Nonexistent Card") → None

    Lookup by face name:
    - get_face_by_name("Stomp") → Some(&CardFace) (adventure face of Bonecrusher Giant)
    - get_face_by_name("Fire") → Some(&CardFace) (first face of Fire // Ice)
    - get_face_by_name("stomp") → Some(&CardFace) (case-insensitive)

    Metadata:
    - card_count() → total number of loaded CardRules
    - errors() → list of (path, error) for files that failed to parse
  </behavior>
  <action>
    1. Add `walkdir = "2"` to engine Cargo.toml dependencies.

    2. Create `database/mod.rs` with `pub mod card_db; pub use card_db::CardDatabase;`

    3. Create `database/card_db.rs`:
       - `CardDatabase` struct:
         - `cards: HashMap<String, CardRules>` — lowercased primary name → CardRules
         - `face_index: HashMap<String, CardFace>` — lowercased face name → cloned CardFace for independent lookup
         - `errors: Vec<(PathBuf, String)>` — paths that failed to parse with error message
       - `CardDatabase::load(root: &Path) -> Result<Self, ParseError>`:
         - Use `walkdir::WalkDir::new(root)` to recursively find all .txt files
         - Filter to files with .txt extension
         - Skip dotfiles/dot-directories (any component starting with '.')
         - For each file: read_to_string, call parse_card_file
         - On success: insert into cards HashMap with lowercased name key; for each face name from card_rules.face_names(), insert face clone into face_index
         - On parse error: push (path, error.to_string()) to errors vec, continue to next file
         - Return CardDatabase
       - `get_by_name(&self, name: &str) -> Option<&CardRules>`:
         - Lookup `self.cards.get(&name.to_lowercase())`
       - `get_face_by_name(&self, name: &str) -> Option<&CardFace>`:
         - Lookup `self.face_index.get(&name.to_lowercase())`
       - `card_count(&self) -> usize` — returns cards.len()
       - `errors(&self) -> &[(PathBuf, String)]` — returns parse errors

    4. Update `lib.rs` to add `pub mod database;`

    5. Write tests:
       - Create a temp directory with sample card files (use tempfile or std::fs in test setup)
       - Test loading a directory with 3-4 card files (single + multi-face)
       - Test case-insensitive name lookup
       - Test face-level lookup for multi-face cards
       - Test that parse errors for malformed files don't prevent other cards from loading
       - Test empty directory returns empty database
       - Test card_count() reflects loaded cards
       - Add an `#[ignore]` test that loads real Forge card files from `../forge/forge-gui/res/cardsfolder/` (if the path exists) and reports card_count() and error count — useful for manual verification

    Note: Clone CardFace for face_index — simpler than lifetime references, CardFace structs are small (mostly String fields).
  </action>

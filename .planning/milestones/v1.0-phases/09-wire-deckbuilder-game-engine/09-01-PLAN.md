  <action>
Create `crates/engine/src/game/deck_loading.rs` with:

1. A serializable `DeckEntry` struct: `{ card: CardFace, count: u32 }` with Serialize/Deserialize.
2. A serializable `DeckPayload` struct: `{ player_deck: Vec<DeckEntry>, opponent_deck: Vec<DeckEntry> }` with Serialize/Deserialize.
3. `pub fn create_object_from_card_face(state: &mut GameState, card_face: &CardFace, owner: PlayerId) -> ObjectId` that:
   - Calls `zones::create_object(state, CardId(state.next_object_id), owner, card_face.name.clone(), Zone::Library)` to allocate the object
   - Gets a mutable reference to the created object
   - Populates: `card_types = card_face.card_type.clone()`, `mana_cost = card_face.mana_cost.clone()`
   - Parses power/toughness from `Option<String>` to `Option<i32>` using `.and_then(|s| s.parse::<i32>().ok())`. Variable P/T like "*" defaults to 0 via `.or(Some(0))` only when the string is present but non-numeric.
   - Sets `base_power = power`, `base_toughness = toughness`
   - Parses loyalty from `Option<String>` to `Option<u32>` same pattern
   - Calls `parse_keywords(&card_face.keywords)` for `keywords` and `base_keywords = keywords.clone()`
   - Copies `abilities = card_face.abilities.clone()`, `svars = card_face.svars.clone()`
   - For trigger_definitions: iterate card_face.triggers and use `parse_ability` to get trigger definitions (same pattern as how engine tests set up triggered abilities). If parsing fails, skip the trigger gracefully.
   - For static_definitions: iterate card_face.static_abilities similarly.
   - For replacement_definitions: iterate card_face.replacements similarly.
   - Derives color: use `card_face.color_override` if present, otherwise derive from mana_cost by collecting unique ManaColor values from cost shards
   - Sets `base_color = color.clone()`
   - Returns the ObjectId

4. `pub fn load_deck_into_state(state: &mut GameState, payload: &DeckPayload)` that:
   - For each entry in `player_deck`: calls `create_object_from_card_face` N times (once per count) for `PlayerId(0)`
   - For each entry in `opponent_deck`: calls `create_object_from_card_face` N times for `PlayerId(1)`
   - After all objects created, shuffles each player's library using `state.rng` (use `rand::seq::SliceRandom` with `shuffle(&mut state.rng)`)

Register the module in `crates/engine/src/game/mod.rs`: add `pub mod deck_loading;` and re-export `pub use deck_loading::{load_deck_into_state, DeckPayload, DeckEntry};`.

Create `crates/engine/src/bin/card_data_export.rs` following the same pattern as `coverage_report.rs`:
- Takes card data directory path as CLI arg or `FORGE_CARDS_PATH` env var
- Loads `CardDatabase`
- Iterates all cards, extracts the primary `CardFace` from each `CardRules`
- Serializes as JSON object mapping card name -> CardFace
- Outputs to stdout
- Add the binary target to `Cargo.toml` under `[[bin]]` section (same pattern as coverage-report)

Add unit tests in deck_loading.rs:
- Test `create_object_from_card_face` with a manually-constructed CardFace produces a GameObject with correct card_types, power/toughness (numeric), keywords, mana_cost, and color
- Test `load_deck_into_state` with a small payload creates correct number of objects in each player's library
- Test that libraries are shuffled (with a seeded RNG, verify library order differs from insertion order for decks with enough cards)
- Test power/toughness "*" values parse to 0

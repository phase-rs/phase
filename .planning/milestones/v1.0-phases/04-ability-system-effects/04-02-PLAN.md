**Create game/targeting.rs:**
- `pub fn find_legal_targets(state: &GameState, filter: &str, source_controller: PlayerId, source_id: ObjectId) -> Vec<TargetRef>` -- parses ValidTgts$ string and returns all legal targets:
  - `"Any"` -- all creatures on battlefield + all players
  - `"Creature"` -- all creatures on battlefield
  - `"Creature.YouCtrl"` -- creatures you control
  - `"Creature.OppCtrl"` -- creatures opponent controls
  - `"Creature.nonBlack"` (and similar `.non{Color}` filters) -- creatures not of that color
  - `"Player"` -- all players
  - `"Card"` with `TargetType$ Spell` in params -- all spells on the stack (by StackEntry id)
  - For each candidate, check hexproof (can't be targeted by opponents) and shroud (can't be targeted by anyone). Check via obj.keywords containing "Hexproof" or "Shroud" strings.
- `pub fn validate_targets(state: &GameState, targets: &[TargetRef], filter: &str, source_controller: PlayerId, source_id: ObjectId) -> Vec<TargetRef>` -- rechecks targets on resolution, returns only still-legal targets. Used for fizzle check.
- `pub fn check_fizzle(original_targets: &[TargetRef], legal_targets: &[TargetRef]) -> bool` -- returns true if ALL original targets are now illegal (spell fizzles per rule 608.2b).

**Create game/casting.rs:**
- `pub fn handle_cast_spell(state: &mut GameState, player: PlayerId, card_id: CardId, events: &mut Vec<GameEvent>) -> Result<WaitingFor, EngineError>`:
  1. Find object in player's hand matching card_id
  2. Get the first ability string from `obj.abilities[0]` and parse with `parse_ability()`
  3. Validate timing: if ability kind is Spell and card has no "Flash" keyword and no Instant core type, require main phase + empty stack + active player (sorcery-speed check). Instants and flash can be cast at any priority.
  4. Build ResolvedAbility from AbilityDefinition, attaching source_id, controller, and svars from card data (obj's raw data -- for now use empty svars, Plan 03 wires card face svars)
  5. Check ValidTgts$ param -- if present, call find_legal_targets():
     - If 0 legal targets -> return error (can't cast)
     - If exactly 1 legal target -> auto-select (set in resolved.targets)
     - If multiple -> return WaitingFor::TargetSelection with PendingCast
  6. Pay mana cost using existing mana_payment::can_pay and mana_payment::pay_cost. If ManaCost contains X shards, do NOT auto-pay (return WaitingFor::ManaPayment for player input). Otherwise auto-pay.
  7. Move card from hand to stack zone, push StackEntry with StackEntryKind::Spell { card_id, ability }
  8. Reset priority_pass_count, emit SpellCast event, return WaitingFor::Priority

- `pub fn handle_select_targets(state: &mut GameState, player: PlayerId, targets: Vec<TargetRef>, events: &mut Vec<GameEvent>) -> Result<WaitingFor, EngineError>`:
  1. Extract PendingCast from WaitingFor::TargetSelection
  2. Validate submitted targets are legal (call find_legal_targets, check targets are subset)
  3. Set targets on the ability, proceed to cost payment (auto-pay or WaitingFor::ManaPayment)
  4. Push to stack, return WaitingFor::Priority

- `pub fn handle_activate_ability(state: &mut GameState, player: PlayerId, source_id: ObjectId, ability_index: usize, events: &mut Vec<GameEvent>) -> Result<WaitingFor, EngineError>`:
  1. Find object on battlefield, verify controller == player
  2. Parse ability from obj.abilities[ability_index]
  3. Handle tap cost: if ability params contain `Cost$ T` or cost string starts with "T", check obj.tapped == false, then tap it
  4. Handle targeting and mana cost similar to spell casting
  5. Push StackEntry with StackEntryKind::ActivatedAbility
  6. Return WaitingFor::Priority

**Update game/stack.rs -- resolve_top with effect execution:**
- Modify `resolve_top` to accept the effect registry: `pub fn resolve_top(state: &mut GameState, events: &mut Vec<GameEvent>, registry: &HashMap<String, EffectHandler>)`
- Before moving the card to its destination zone, execute the effect:
  1. Pop the top StackEntry
  2. Extract the ResolvedAbility from the StackEntryKind (Spell or ActivatedAbility)
  3. Run fizzle check: call `targeting::validate_targets()` to get still-legal targets. If check_fizzle returns true, move card to graveyard without executing effects, emit StackResolved event, return early.
  4. Update the ability's targets to only the still-legal ones
  5. Call `effects::resolve_effect(registry, state, &ability, events)` to execute the effect
  6. Determine destination zone (permanent types -> battlefield, others -> graveyard, activated abilities -> no zone movement since source stays on battlefield)
  7. Move card to destination zone
  8. Emit StackResolved event
- Update all existing callers of resolve_top to pass the registry. The registry should be built once and stored. For simplicity, build it in the apply function or pass it through. Recommended: build_registry() at the top of apply() or use a lazy_static/once pattern. Since this is called per-action and build_registry is cheap (15 inserts), building each time in apply() is fine for now.

**Update game/engine.rs -- add CastSpell, ActivateAbility, SelectTargets arms:**
- Add match arm for `(WaitingFor::Priority { player }, GameAction::CastSpell { card_id, .. })`:
  - Validate priority_player == player
  - Call `casting::handle_cast_spell(state, player, card_id, &mut events)`
- Add match arm for `(WaitingFor::Priority { player }, GameAction::ActivateAbility { source_id, ability_index })`:
  - Validate priority_player == player
  - Call `casting::handle_activate_ability(state, player, source_id, ability_index, &mut events)`
- Add match arm for `(WaitingFor::TargetSelection { player, .. }, GameAction::SelectTargets { targets })`:
  - Call `casting::handle_select_targets(state, player, targets, &mut events)`
- Update the PassPriority handler: when both players pass and stack is NOT empty, resolve top of stack (calling resolve_top with registry) then give priority to active player. This replaces the current behavior where the stack was resolved but effects were not executed.
- Remove the old test `apply_rejects_unsupported_actions` that tests CastSpell being rejected, since CastSpell is now supported.
- Build the effect registry at the start of apply() with `let registry = effects::build_registry();`
- Update imports to include casting, targeting, effects modules.

    **evaluate_layers()** (new file `crates/engine/src/game/layers.rs`):
    Implement the full layer evaluation pipeline:

    1. `pub fn evaluate_layers(state: &mut GameState)`:
       - Step 1: Reset all battlefield objects' computed characteristics to base values. For each object on battlefield: `power = base_power`, `toughness = base_toughness`, `keywords = base_keywords.clone()`, `color = base_color.clone()`. Also reset card_types to base (need base_card_types field -- add to GameObject if not present, or just don't reset types since type changes are rare and complex).
       - Step 2: Call `gather_active_continuous_effects(state)` to collect all active effects.
       - Step 3: For each layer in Layer::all(), filter effects to that layer, order them (dependency if applicable, else timestamp), then apply each.
       - Step 4: Apply counter-based P/T (layer 7e) -- iterate battlefield objects, sum +1/+1 and -1/-1 counters into power/toughness.
       - Set `state.layers_dirty = false`.

    2. `fn gather_active_continuous_effects(state: &GameState) -> Vec<ActiveContinuousEffect>`:
       - Scan all objects in battlefield (and command zone for emblems) that have non-empty static_definitions.
       - For each static_definition with mode "Continuous", determine which layer(s) it affects based on params (AddPower/AddToughness -> ModifyPT, SetPower/SetToughness -> SetPT, AddKeyword/RemoveKeyword -> Ability, AddType -> Type, SetColor/AddColor -> Color, etc.).
       - Create an ActiveContinuousEffect for each (source, definition, layer) tuple.

    3. `fn order_with_dependencies(effects: &[ActiveContinuousEffect], state: &GameState) -> Vec<ActiveContinuousEffect>`:
       - Sort by deterministic fallback first: (timestamp, source_id.0, def_index) per research recommendation.
       - Build petgraph DiGraph with effects as nodes.
       - For each pair (i, j), check if i depends on j using `depends_on()`. If so, add edge j -> i.
       - Call `petgraph::algo::toposort()`. On Ok, return toposorted order. On Err (cycle), fall back to timestamp ordering per CR 613.8.

    4. `fn depends_on(a: &ActiveContinuousEffect, b: &ActiveContinuousEffect, _state: &GameState) -> bool`:
       - If b changes types (layer Type) and a's affected_filter is type-based (e.g., "Creature"), a depends on b.
       - If b adds/removes abilities (layer Ability) and a checks for abilities, a depends on b.
       - Conservative initial implementation: check if b's layer could affect a's filter criteria.

    5. `fn order_by_timestamp(effects: &[ActiveContinuousEffect]) -> Vec<ActiveContinuousEffect>`:
       - Sort by (timestamp, source_id.0, def_index) ascending.

    6. `fn apply_continuous_effect(state: &mut GameState, effect: &ActiveContinuousEffect)`:
       - Determine affected objects by evaluating effect.affected_filter against battlefield (use card_matches_filter or similar logic).
       - Apply the modification based on effect.mode and params to each affected object's computed fields.

    Register `pub mod layers;` in `game/mod.rs`. Export `pub use layers::evaluate_layers;`.

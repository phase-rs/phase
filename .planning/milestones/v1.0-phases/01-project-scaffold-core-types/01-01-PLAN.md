  </files>
  <action>
    Create the Cargo workspace root:
    - `Cargo.toml` with `[workspace]`, `resolver = "2"`, `members = ["crates/*"]`
    - Workspace dependencies: `serde = { version = "1", features = ["derive"] }`, `rpds = { version = "1.2" }`
    - Release profile optimized for WASM size: `opt-level = 'z'`, `lto = true`, `codegen-units = 1`, `strip = true`, `panic = 'abort'`

    Create `.cargo/config.toml` with alias for WASM build convenience.

    Create `crates/engine/Cargo.toml`:
    - Library crate (default lib type)
    - Dependencies: `serde.workspace = true`, `rpds.workspace = true`

    Create `crates/engine/src/lib.rs`:
    - `pub mod types;` (module declaration only)

    Create `crates/engine-wasm/Cargo.toml`:
    - `crate-type = ["cdylib"]`
    - Dependencies: `engine = { path = "../engine" }`, `wasm-bindgen = "0.2.114"`, `tsify = { version = "0.5", features = ["js"] }`, `serde = { version = "1", features = ["derive"] }`, `serde-wasm-bindgen = "0.6"`
    - NOTE: Use `tsify` (NOT tsify-next -- it is unmaintained per RUSTSEC-2025-0048)

    Create `crates/engine-wasm/src/lib.rs`:
    - `use wasm_bindgen::prelude::*;`
    - A `ping()` function exported via `#[wasm_bindgen]` returning `"forge-rs engine ready"`
    - A `create_initial_state()` function that creates a default `GameState` and returns it via `serde_wasm_bindgen::to_value`
    - Tsify re-exports for core types (GameAction, GameEvent, GameState, Zone, Phase, ManaColor, ManaPool) using newtype wrapper pattern with `#[derive(Tsify)]` and `#[tsify(into_wasm_abi, from_wasm_abi)]`

    Create `scripts/build-wasm.sh`:
    - Three-step pipeline: (1) `cargo build --package engine-wasm --target wasm32-unknown-unknown` (2) `wasm-bindgen --target web --out-dir client/src/wasm --out-name engine_wasm` (3) `wasm-opt -Oz` (release only)
    - Accept profile argument (default: release)
    - Print binary size at end
    - Make executable: `chmod +x`

    Create `.gitignore` with Rust (`/target/`), Node (`node_modules/`), WASM output (`client/src/wasm/`), and OS entries.

    Ensure `wasm32-unknown-unknown` target is installed: `rustup target add wasm32-unknown-unknown`
  </action>
  <verify>
    <automated>cd /Users/matt/dev/forge.rs && cargo build 2>&1 && cargo build --package engine-wasm --target wasm32-unknown-unknown 2>&1 && echo "Both targets compile"</automated>
  </verify>
  <done>Cargo workspace compiles for both native and wasm32-unknown-unknown targets. engine-wasm crate has wasm_bindgen exports.</done>

  <action>
    Create type modules in `crates/engine/src/types/`. Write tests FIRST (in `#[cfg(test)] mod tests` within each file), then implement to make them pass.

    `identifiers.rs`:
    - `CardId(u64)` -- newtype for card identity (definition-level)
    - `ObjectId(u64)` -- newtype for game object identity (instance-level)
    - Both derive: Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize

    `mana.rs`:
    - `ManaColor` enum: White, Blue, Black, Red, Green -- derives Serialize/Deserialize
    - `ManaPool` struct with fields for each color + colorless (u32 each), derive Default, Serialize, Deserialize
    - Implement `ManaPool::add(&mut self, color: ManaColor, amount: u32)` and `ManaPool::total(&self) -> u32`

    `phase.rs`:
    - `Phase` enum with all MTG turn phases: Untap, Upkeep, Draw, PreCombatMain, BeginCombat, DeclareAttackers, DeclareBlockers, CombatDamage, EndCombat, PostCombatMain, End, Cleanup
    - `Step` can be folded into Phase variants or kept separate -- use judgment for cleanest MTG modeling
    - Use `#[serde(tag = "type")]` for tagged serialization

    `zones.rs`:
    - `Zone` enum: Library, Hand, Battlefield, Graveyard, Stack, Exile, Command
    - Use `#[serde(tag = "type")]` for tagged serialization

    `card.rs`:
    - `CardDefinition` stub struct with minimal fields: name (String), mana_cost (Option<String>), type_line (String), oracle_text (Option<String>)
    - This is a placeholder -- Phase 2 parser fills in the real structure
    - Derive Serialize, Deserialize, Debug, Clone

    `player.rs`:
    - `PlayerId(u8)` newtype
    - `Player` struct: id (PlayerId), life (i32 -- starts at 20), mana_pool (ManaPool)
    - Derive Default for Player with sensible starting values

    `actions.rs`:
    - `GameAction` enum with `#[serde(tag = "type", content = "data")]`:
      PassPriority, PlayLand { card_id: CardId }, CastSpell { card_id: CardId, targets: Vec<ObjectId> }, ActivateAbility { source_id: ObjectId, ability_index: usize }, DeclareAttackers { attacker_ids: Vec<ObjectId> }, DeclareBlockers { assignments: Vec<(ObjectId, ObjectId)> }, MulliganDecision { keep: bool }
    - Phase 1 defines top-level variants; later phases add data

    `events.rs`:
    - `GameEvent` enum with `#[serde(tag = "type", content = "data")]`:
      GameStarted, TurnStarted { player_id: PlayerId, turn_number: u32 }, PhaseChanged { phase: Phase }, PriorityPassed { player_id: PlayerId }, SpellCast { card_id: CardId, controller: PlayerId }, AbilityActivated { source_id: ObjectId }, ZoneChanged { object_id: ObjectId, from: Zone, to: Zone }, LifeChanged { player_id: PlayerId, amount: i32 }, ManaAdded { player_id: PlayerId, color: ManaColor, amount: u32 }, GameOver { winner: Option<PlayerId> }
    - These are the top-level variants; later phases add more

    `game_state.rs`:
    - `GameState` struct: turn_number (u32), active_player (PlayerId), phase (Phase), players (Vec<Player>), priority_player (PlayerId)
    - Derive Default with 2-player game setup (turn 0, untap phase)

    `mod.rs`:
    - Re-export all submodules and key types at module level

    - `submitAction()`: validates initialized, queues action, processes queue sequentially (WASM is single-threaded)
    - `getState()`: calls WASM `create_initial_state()` (placeholder until Phase 3 adds real state)
    - `dispose()`: resets state
    - Error normalization: catch WASM errors, wrap in `AdapterError` with code, message, recoverable fields
    - Async queue pattern per CONTEXT.md decision: serialized access to WASM

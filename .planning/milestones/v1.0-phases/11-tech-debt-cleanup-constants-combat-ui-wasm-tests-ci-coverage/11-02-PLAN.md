   - Add to UiStoreState: `combatMode: 'attackers' | 'blockers' | null`, `selectedAttackers: number[]` (ObjectId[]), `blockerAssignments: Map<number, number>` (blocker ObjectId -> attacker ObjectId)
   - Add to UiStoreActions: `setCombatMode(mode)`, `toggleAttacker(id)`, `selectAllAttackers(ids)`, `assignBlocker(blockerId, attackerId)`, `removeBlockerAssignment(blockerId)`, `clearCombatSelection()`
   - Initialize: `combatMode: null`, `selectedAttackers: []`, `blockerAssignments: new Map()`
   - `toggleAttacker`: if already in selectedAttackers, remove it; else add it
   - `clearCombatSelection`: reset all three combat fields to defaults

2. Create `client/src/components/combat/AttackerControls.tsx`:
   - Props: `{ onAttackAll: () => void; onSkip: () => void; onConfirm: () => void; attackerCount: number }`
   - Three buttons in a horizontal row, positioned at bottom-center of the board area
   - "Attack All" (amber/orange), "Skip" (gray), "Confirm Attackers" (green, shows count)
   - Tailwind styling matching existing control button patterns

3. Create `client/src/components/combat/BlockerControls.tsx`:
   - Props: `{ onConfirm: () => void; assignmentCount: number }`
   - "Confirm Blockers" button (green, shows assignment count)
   - Positioned at bottom-center like AttackerControls

4. Create `client/src/components/combat/BlockerArrow.tsx`:
   - Reuse the TargetArrow pattern but with red/orange color scheme (not cyan)
   - Props: `{ blockerId: number; attackerId: number }` — looks up DOM positions via `document.querySelector([data-object-id="..."])` and `getBoundingClientRect()`
   - If either element not found, render nothing
   - Arrow color: `rgba(249,115,22,0.8)` (orange-500) for stroke, matching attacker glow theme

5. Create `client/src/components/combat/CombatOverlay.tsx`:
   - Props: `{ mode: 'attackers' | 'blockers' }`
   - On mount: call `setCombatMode(mode)` from uiStore
   - On unmount: call `clearCombatSelection()`
   - **Attacker mode:**
     - Read `gameState.objects` to find creatures controlled by the human player (player 0) that are on the battlefield and not tapped (valid attackers). The engine sends DeclareAttackers with `player: PlayerId` — use that to identify which player's creatures.
     - Clicking a creature toggles it via `toggleAttacker(objectId)` — PermanentCard will handle the click delegation (Task 2)
     - Render AttackerControls with handlers:
       - Attack All: `selectAllAttackers(allValidAttackerIds)`
       - Skip: dispatch `{ type: "DeclareAttackers", data: { attacker_ids: [] } }` and clearCombatSelection
       - Confirm: dispatch `{ type: "DeclareAttackers", data: { attacker_ids: selectedAttackers } }` and clearCombatSelection
   - **Blocker mode:**
     - Track a `pendingBlocker: ObjectId | null` state (the creature being assigned as a blocker)
     - First click on a valid blocker sets pendingBlocker; second click on an attacker completes the assignment via `assignBlocker(blockerId, attackerId)`
     - Render BlockerArrow for each entry in blockerAssignments
     - Render BlockerControls with handler:
       - Confirm: dispatch `{ type: "DeclareBlockers", data: { assignments: Array.from(blockerAssignments.entries()) } }` and clearCombatSelection
   - Use z-30 for the controls overlay to stay below modals (z-50) but above the board

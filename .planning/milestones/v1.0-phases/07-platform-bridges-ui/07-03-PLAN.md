  <action>
1. Create `client/src/hooks/useLongPress.ts`:
   - Custom hook: `useLongPress(callback: () => void, options?: { delay?: number })`.
   - Returns event handlers: onTouchStart, onTouchEnd, onTouchCancel (cancel on scroll).
   - Default delay 500ms. Clears timeout on touchEnd/touchCancel.
   - Only fires on touch devices (no-op on mouse -- mouse uses hover).

2. Create `client/src/components/board/PermanentCard.tsx`:
   - Props: `{ objectId: number }`.
   - Reads GameObject from gameStore via `useGameStore(s => s.gameState?.objects[objectId])`.
   - Reads UI state from uiStore (selected, hovered, targeting).
   - Renders CardImage (from Plan 02) with the object's card name.
   - Tap state: tapped prop on CardImage (30-degree rotation).
   - Counters: render as small numbered badges (e.g., "+1/+1 x3") positioned at bottom-right of card. Each counter type gets a colored badge.
   - Attachments: render attached cards slightly offset behind (translate-y by -10px per attachment, z-indexed so host is on top).
   - Damage overlay: if damage_marked > 0, show red semitransparent overlay at bottom with damage number.
   - Summoning sickness: if `entered_battlefield_turn === gameState.turn_number` and object is a creature and doesn't have haste keyword, apply desaturated filter (`filter: saturate(0.5)`) and faint glow ring (`box-shadow: 0 0 6px 1px rgba(255,255,255,0.3)`).
   - Interactability glow: white glow ring when card can be activated/targeted (driven by uiStore).
   - Target glow: cyan glow ring when card is a valid target during targeting mode.
   - Click handler: dispatch to uiStore (select object, or add target if in targeting mode).
   - Long-press: opens card inspection via uiStore.inspectObject.
   - Hover: sets uiStore.hoveredObjectId.

3. Create `client/src/components/board/BattlefieldRow.tsx`:
   - Props: `{ objectIds: number[], rowType: 'creatures' | 'lands' | 'other' }`.
   - Renders a horizontal flex row of PermanentCards.
   - Uses Framer Motion `layoutId` on each card for smooth repositioning animations.
   - Flex wrap for overflow, gap between cards.

4. Create `client/src/components/board/GameBoard.tsx`:
   - Reads gameState from gameStore.
   - Partitions battlefield objects by controller (player 0 vs player 1) and by type (creatures, lands, other permanents) based on card_types.
   - Layout (Tailwind grid/flex):
     - Top: opponent's battlefield (BattlefieldRows: other, creatures, lands -- top to bottom from opponent perspective)
     - Middle: shared space (stack/controls will go here in Plan 04, just a flex spacer for now)
     - Bottom: player's battlefield (BattlefieldRows: lands, creatures, other -- bottom to top from player perspective)
   - Full height: h-screen with flex-col, each section taking appropriate space.
   - Dark background matching Arena aesthetic (bg-gray-950).
  </action>

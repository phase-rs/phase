  <behavior>
    cardProps.ts - toCardProps(obj: GameObject) -> CardViewProps:
    - Maps id, name, tapped, power, toughness, basePower, baseToughness, damageMarked
    - isPowerBuffed: true when power > base_power
    - isPowerDebuffed: true when power < base_power
    - isToughnessBuffed: true when toughness > base_toughness
    - isToughnessDebuffed: true when toughness < base_toughness OR damage_marked > 0
    - effectiveToughness: toughness - damage_marked (null if toughness null)
    - Extracts counters as Array<{ type: string; count: number }>
    - isCreature/isLand from card_types.core_types
    - attachedTo, attachmentIds, keywords, colorIdentity

    cardProps.ts - computePTDisplay(obj: GameObject) -> PTDisplay | null:
    - Returns null for non-creatures (power/toughness null)
    - power color: green when buffed, red when debuffed, white when base
    - toughness color: red when damage_marked > 0, green when buffed, red when debuffed, white when base

    battlefieldProps.ts - partitionByType(objects: GameObject[]):
    - Separates into creatures, lands, other arrays (by ObjectId)
    - Creatures: core_types includes "Creature"
    - Lands: core_types includes "Land"
    - Other: everything else (artifacts, enchantments, planeswalkers)

    battlefieldProps.ts - groupByName(objects: GameObject[]) -> GroupedPermanent[]:
    - Groups same-name permanents into stacks with count badge
    - Only groups if ALL share: same name, same tapped state, no attachments, no counters
    - Permanents with unique state (attachments, counters, different tap) shown individually
    - Returns representative (first) object + array of all ids + count

    dominantColor.ts - getDominantManaColor(battlefieldIds, objects, playerId):
    - Filters to lands controlled by playerId
    - Counts color identity occurrences
    - Returns the most common ManaColor, or null if no colored lands

    logFormatting.ts - classifyEventColor(event: GameEvent) -> string:
    - combat events (AttackersDeclared, BlockersDeclared, DamageDealt) -> "red"
    - spell events (SpellCast, StackPushed, StackResolved, SpellCountered) -> "blue"
    - life events (LifeChanged) -> amount >= 0 ? "green" : "red"
    - zone events (ZoneChanged, Discarded) -> "gray"
    - default -> "gray"

    logFormatting.ts - filterByVerbosity(events: GameEvent[], level: LogVerbosity) -> GameEvent[]:
    - "full": all events
    - "compact": exclude PriorityPassed, ManaAdded, PermanentTapped/Untapped, DamageCleared
    - "minimal": only GameStarted, TurnStarted, SpellCast, DamageDealt, LifeChanged, GameOver, AttackersDeclared, BlockersDeclared, CreatureDestroyed, TokenCreated
  </behavior>

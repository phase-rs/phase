**client/src/game/autoPass.ts** — Create auto-pass heuristic module:
- Export `function shouldAutoPass(state: GameState, waitingFor: WaitingFor, phaseStops: Phase[], fullControl: boolean): boolean`
- Rules (per user decisions — MTGA-style aggressive auto-pass):
  1. If `fullControl` is true, return false (never auto-pass)
  2. If `waitingFor.type !== "Priority"`, return false
  3. If `waitingFor.data.player !== PLAYER_ID`, return false (only auto-pass for local player)
  4. If `state.stack.length > 0`, return false (never auto-pass with items on stack — opponent may have cast something)
  5. If current phase is in `phaseStops`, return false
  6. If player has relevant instant-speed actions (see `hasRelevantActions` below), return false
  7. Otherwise, return true
- Internal `function hasRelevantActions(state: GameState): boolean`:
  - Get player via `state.players[PLAYER_ID]`
  - Check if player has mana available: `player.mana_pool.mana.length > 0`
  - Check if player has instant-speed cards in hand: iterate `player.hand`, look up each `state.objects[id]`, check if `card_types.core_types.includes("Instant")` or `keywords.includes("Flash")`
  - Return `hasMana && hasInstants` — conservative heuristic (stops if unsure)
  - Also check for activated abilities on battlefield permanents: if any player-controlled permanent has non-empty `abilities` array and player has mana, be conservative and return true. This catches edge cases like activated abilities.
  - NOTE: This is a client-side approximation. The heuristic is intentionally conservative (stops when unsure) to avoid frustrating players. Research recommended starting with this and adding WASM `get_legal_actions` binding later if needed.
- Import PLAYER_ID from constants/game, types from adapter/types

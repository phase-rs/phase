**client/src/components/controls/PhaseStopBar.tsx** — Clickable phase indicator strip with stop toggles:

1. Define `ALL_PHASES` array of all 12 Phase values in order: Untap, Upkeep, Draw, PreCombatMain, BeginCombat, DeclareAttackers, DeclareBlockers, CombatDamage, EndCombat, PostCombatMain, End, Cleanup

2. Define `PHASE_LABELS` mapping Phase to short display labels: UT, UP, DR, M1, BC, DA, DB, CD, EC, M2, EN, CL

3. Component reads:
   - `currentPhase` from `useGameStore(s => s.gameState?.phase)` — to highlight active phase
   - `phaseStops` from `usePreferencesStore(s => s.phaseStops)` — which phases are set to stop
   - `setPhaseStops` from `usePreferencesStore(s => s.setPhaseStops)`

4. Render a horizontal strip of phase buttons:
   - Each button shows the short label (e.g., "M1" for PreCombatMain)
   - Click toggles the phase in/out of `phaseStops` array
   - Visual states:
     - **Active phase**: bright text (white) with a colored bottom border (cyan-400) — shows where the game currently is
     - **Stop set (lit)**: amber/yellow dot indicator below the label — player will pause here
     - **No stop (dim)**: gray text, no indicator — auto-pass will skip
   - Tooltip on hover showing full phase name (use `title` attribute for simplicity)

5. Layout: `flex items-center gap-0.5` with each button being a small `px-1.5 py-0.5 text-[10px]` element. The strip should be compact enough to fit in the center divider of GamePage.

**client/src/pages/GamePage.tsx** — Major simplification:

1. **Extract game initialization into GameProvider:**
   - Move the adapter creation logic (WasmAdapter/WebSocketAdapter), deck loading (`loadActiveDeck`, `buildDeckPayload`), and `initGame` calls into GameProvider
   - GameProvider already manages the game loop controller lifecycle (from Plan 02)
   - Update GameProvider props to accept the full initialization config: `{ mode, difficulty, joinCode, children }`
   - GameProvider's useEffect handles: create adapter, load deck, call initGame, create game loop controller, start it, return cleanup that disposes everything

2. **GamePage becomes layout-only:**
   - Read mode/difficulty/joinCode from URL params (same as now)
   - Wrap content in `<GameProvider mode={mode} difficulty={difficulty} joinCode={joinCode}>`
   - Remove the entire ~100-line useEffect that creates adapters, loads decks, inits games, creates AI controllers
   - Remove `aiControllerRef`, `wsAdapterRef` refs — GameProvider owns these now
   - Keep: all the UI rendering (board, hand, HUD, overlays, modals)
   - Keep: mulligan handlers, zone viewer state, preferences modal state — these are UI concerns
   - Replace `useGameDispatch()` calls with `useDispatch()` from GameProvider context
   - The online multiplayer state (hostGameCode, waitingForOpponent, opponentDisconnected, etc.) needs to stay in GamePage for now since it drives UI overlays. GameProvider can expose WS events through a callback or the state can be lifted later.

3. **Replace phase indicator with PhaseStopBar:**
   - In the center divider section, replace the plain text `{gameState?.phase ?? "---"}` with `<PhaseStopBar />`
   - Keep the turn number display alongside it

4. **Estimated size reduction:** GamePage should drop from ~650 lines to ~400-450 lines. The biggest savings come from removing the adapter/init useEffect (~100 lines) and AI controller management (~20 lines). The helper functions (loadActiveDeck, buildDeckPayload, parsedDeckToDeckData, getWsUrl) move to GameProvider or a shared utils module.

NOTE: The online multiplayer state management (WS adapter events, reconnection state) is the trickiest part. Two approaches:
- **Option A (simpler):** Keep WS event handling in GamePage — GameProvider creates the adapter and passes it back via a ref/callback so GamePage can subscribe to WS events for its overlays. This keeps the diff smaller.
- **Option B (cleaner):** Move all WS state to a Zustand store or GameProvider context. More work but better separation.
- Use Option A for this plan — keep WS overlay state in GamePage with minimal changes. A future cleanup can move it.

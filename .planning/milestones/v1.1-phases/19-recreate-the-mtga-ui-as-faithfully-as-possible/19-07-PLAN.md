**SplashScreen.tsx — new component:**

Props: `{ progress: number; onComplete: () => void }` where progress is 0-100.

Layout:
1. Full-screen dark background (bg-gray-950).
2. Centered Forge.rs logo (`logo.webp` from public/) at ~200px width.
3. Below logo: "Forge.rs" text in subtle gray.
4. Loading bar: thin horizontal bar (~200px wide, 4px tall) with animated fill. Fill color: indigo-500.
5. Below bar: "Loading..." text that updates to "Ready" at 100%.
6. When progress reaches 100, fade out the entire splash screen (0.5s fade) and call onComplete.

Use Framer Motion for logo entrance (fade+scale from 0.8 to 1.0) and splash exit (fade out).

**MenuParticles.tsx — new component:**

Canvas-based animated particle background for the menu.

Implementation:
1. Full-screen canvas with `pointer-events: none`, positioned behind menu content.
2. Generate 40-60 floating particles:
   - Random positions across the canvas
   - Small size: 2-4px radius
   - Colors: mix of indigo, cyan, amber at low opacity (0.1-0.3)
   - Slow upward drift: 10-30px/s
   - Slight horizontal oscillation: sin(time * frequency) * amplitude
   - When a particle drifts off top, respawn at bottom with new random x
3. Use requestAnimationFrame loop.
4. Clean up on unmount (cancel animation frame).

**DeckGallery.tsx — new component:**

Props: `{ onSelectDeck: (deckName: string) => void; activeDeckName: string | null; mode: 'ai' | 'online'; difficulty: string; onDifficultyChange: (d: string) => void; onStartGame: () => void; onBack: () => void }`

Layout:
1. Title area: "Select Deck" centered at top.
2. If mode is 'ai': inline segmented control for difficulty (Easy/Medium/Hard/VeryEasy/VeryHard) below the title. Use a horizontal button group with the active difficulty highlighted.
3. Deck grid: responsive grid of deck tiles. Each tile:
   - Representative card art (Scryfall art_crop) as background. Use the first non-land card name in the deck list to fetch art. Show loading shimmer until loaded.
   - Deck name overlaid at bottom in white text on dark gradient overlay.
   - Color dots (WUBRG) in a small row.
   - Card count badge.
   - Active deck gets a ring-2 ring-indigo-500 highlight.
   - Click to select.
4. "Start Game" / "Host Game" button at bottom (depends on mode), disabled until deck selected.
5. "Back" button to return to mode selection.

For the representative card art: parse the deck from localStorage, find the first non-land card name (check if type includes "Land" from STARTER_DECKS data or just skip basic land names: Plains, Island, Swamp, Mountain, Forest). Use `useCardImage(cardName, { size: 'art_crop' })` for the art tile.

Read saved decks from localStorage using the same pattern as current MenuPage (STORAGE_KEY_PREFIX).

     - "Play Online" — emerald bg
     - "Deck Builder" — outline border style
   - If there's an active saved game: "Resume Game" button above the others (amber bg)

**TargetArrow.tsx — complete rewrite:**

Replace the straight line with a quadratic Bezier curve in gold color.

```typescript
function getArcPath(from: Point, to: Point): string {
  const mx = (from.x + to.x) / 2;
  const my = (from.y + to.y) / 2;
  const dx = to.x - from.x;
  const dy = to.y - from.y;
  const dist = Math.sqrt(dx * dx + dy * dy);
  // Perpendicular offset for curve — proportional to distance
  const offset = Math.min(80, dist * 0.3);
  const nx = -dy / dist;
  const ny = dx / dist;
  const cx = mx + nx * offset;
  const cy = my + ny * offset;
  return `M ${from.x} ${from.y} Q ${cx} ${cy} ${to.x} ${to.y}`;
}
```

Gold color: `#C9B037` (same as multicolor border gold).

SVG rendering:
- Add a glow filter in `<defs>`: `<filter id="gold-glow">` with `feGaussianBlur stdDeviation="3"` and `feMerge` combining blur + original.
- Use `<path d={getArcPath(from, to)}` with `stroke="#C9B037" strokeWidth={3} fill="none" filter="url(#gold-glow)"`.
- Update the arrowhead marker: change fill from cyan to `#C9B037`.
- For non-minimal VFX: animate `pathLength` from 0 to 1 via framer-motion `motion.path`.
- For minimal VFX: static path, no animation.

**TargetingOverlay.tsx:**

If TargetingOverlay passes any color/style props to TargetArrow, update them. Otherwise TargetArrow handles its own styling. Verify the TargetingOverlay still renders correctly after the TargetArrow changes.

**Attacker slide-forward animation:**

When a creature is attacking (`isAttacking` is true), it should slide forward toward the center of the board.

In the `animate` prop of the `motion.div`, add a `y` translation when attacking:
```typescript
animate={{
  rotate: isAttacking || obj.tapped ? 17 : 0,
  opacity: obj.tapped && !isAttacking ? 0.85 : 1,
  y: isAttacking ? -30 : 0,  // Slide up (toward center) when attacking
}}
```

The `-30` value slides the card 30px toward the center (upward for player's creatures). For opponent's creatures the direction should be inverted. Determine if the card belongs to the player (controller === 0) or opponent (controller === 1):
```typescript
const slideDirection = obj.controller === 0 ? -30 : 30;
// ...
y: isAttacking ? slideDirection : 0,
```

This creates the MTGA effect of attackers stepping forward into the "red zone."

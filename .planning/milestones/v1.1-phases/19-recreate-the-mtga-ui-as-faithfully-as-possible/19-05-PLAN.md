**TurnBanner.tsx — complete rewrite:**

Replace the simple slide-in banner with a layered cinematic effect. Total duration: 1.5s.

Phases (overlapping Framer Motion animations):
1. **Light burst** (0-0.3s): Radial gradient circle expands from center, fades. Use a `motion.div` with `radial-gradient(circle, rgba(255,255,255,0.3), transparent)` scaling from 0 to 3.
2. **Banner strip** (0.1-0.4s): Horizontal bar slides in from left across full width. Height ~60px, background matching theme color with 80% opacity.
3. **Diamond accents** (0.3-0.5s): Two diamond shapes appear at left and right edges of the banner strip. Use rotated squares (`rotate-45`) with border only.
4. **Text punch** (0.3-0.6s): "YOUR TURN" / "THEIR TURN" text scales from 0.5 to 1.0 with triple text-shadow glow effect. Use three layered text-shadows at increasing blur radii.
5. **Hold** (0.6-1.2s): Everything visible, slight pulse on text.
6. **Exit** (1.2-1.5s): Everything slides right + fades out simultaneously.

Color theming:
- Your Turn: amber theme — banner `bg-amber-500/80`, text glow amber (#F59E0B), diamond borders amber
- Their Turn: slate theme — banner `bg-slate-600/80`, text glow slate (#64748B), diamond borders slate

Keep the same props interface: `{ turnNumber, isPlayerTurn, speedMultiplier, onComplete }`.
Use AnimatePresence around the whole effect. Call onComplete when exit animation finishes.

For minimal VFX quality: just the text with a simple fade (no light burst, diamonds, or banner strip).
For reduced VFX: banner strip + text, no light burst or diamonds.

**DeathShatter.tsx — new component:**

Canvas-based card fragment shatter effect.

Props: `{ position: { x: number; y: number; width: number; height: number }; imageUrl: string; onComplete: () => void }`

Implementation:
1. Create a canvas element at the provided position (absolute/fixed positioning).
2. Load the art_crop image (from imageUrl, should already be cached).
3. Create an offscreen canvas, draw the image scaled to the card dimensions.
4. Generate 8-12 polygon fragments using random subdivision:
   - Divide the image into a grid of ~3x4 cells
   - Add random perturbation to cell boundaries for organic-looking fragments
   - Each fragment: a polygon clip region of the source image
5. Animate each fragment with:
   - Random outward velocity from card center (150-300px/s)
   - Slight random rotation velocity (180-360deg/s)
   - Gravity pulling down (200px/s^2)
   - Alpha fading from 1.0 to 0.0 over 0.6s
   - Slight red tint flash at the start (first 0.1s)
6. Use requestAnimationFrame loop. Duration: 0.6s total.
7. At the end, trigger a small particle burst at the card center position (use the existing ParticleCanvas pattern — dispatch a one-shot burst effect or render inline particles).
8. Call onComplete when all fragments have faded.
9. Canvas has `pointer-events: none` and z-index between board and modals.

**CastArcAnimation.tsx — new component:**

Animates a card image flying along an arc path.

Props: `{ from: { x: number; y: number }; to: { x: number; y: number }; cardName: string; mode: 'cast' | 'resolve-permanent' | 'resolve-spell'; onComplete: () => void }`

Implementation using Framer Motion:
1. For `cast` mode (hand → stack): Card image flies from `from` to `to` along a parabolic arc (curves upward). Use `motion.div` with keyframe arrays for x and y:
   - x: [from.x, midX, to.x] (linear interpolation)
   - y: [from.y, midY - arcHeight, to.y] where arcHeight = ~100px
   - scale: [1, 1.1, 1] (slight scale pulse at peak)
   - Add a glow effect (box-shadow) that intensifies at the destination.
   - Duration: 0.4s
   - Card size: ~80x112px (small card representation).

2. For `resolve-permanent` mode (stack → battlefield): Card flies from `to` position (stack) to a target position. Similar arc but shorter duration (0.3s). At arrival: brief glow pulse.

3. For `resolve-spell` mode (instant/sorcery on stack): Card fades out with scale reduction at its current position. Duration: 0.3s.

4. Call onComplete after animation finishes.
5. Use `pointer-events: none` and z-index 45.

Use `useCardImage(cardName, { size: 'small' })` for the flying card thumbnail (lightweight).

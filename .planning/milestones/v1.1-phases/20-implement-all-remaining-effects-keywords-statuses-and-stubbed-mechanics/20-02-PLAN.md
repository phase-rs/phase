  <files>crates/engine/src/game/effects/attach.rs, crates/engine/src/game/effects/mod.rs, crates/engine/src/types/game_state.rs, crates/engine/src/types/actions.rs, crates/engine/src/game/engine.rs</files>
  <behavior>
    - Test: Attach effect sets source.attached_to = target and target.attachments includes source
    - Test: Equip action on equipment creates WaitingFor::EquipTarget with valid creature targets
    - Test: Selecting equip target attaches equipment and returns to Priority
    - Test: Equipment that was attached to creature A can be moved to creature B via re-equip
    - Test: GameAction::Equip only allowed at sorcery speed (main phase, empty stack, active player)
  </behavior>
  <action>
1. Create `crates/engine/src/game/effects/attach.rs`:
   - `resolve_attach(state, ability, events)` -- reads "Object" param (source_id or target), sets `attached_to` on the attachment object to the target, adds attachment's id to target's `attachments` vec. Emit a GameEvent (reuse EffectResolved or add a new Attached event if needed).
   - `resolve_equip(state, ability, events)` -- similar to attach but specifically for equipment equip actions.
2. Register "Attach" in `effects/mod.rs` build_registry: `registry.insert("Attach".to_string(), attach::resolve_attach);`
3. Add `WaitingFor::EquipTarget { player: PlayerId, equipment_id: ObjectId, valid_targets: Vec<ObjectId> }` to game_state.rs WaitingFor enum.
4. Add `GameAction::Equip { equipment_id: ObjectId, target_id: ObjectId }` to actions.rs.
5. In engine.rs, add match arm for `(WaitingFor::EquipTarget { player, equipment_id, .. }, GameAction::Equip { equipment_id: eq_id, target_id })`:
   - Validate target_id is a creature controlled by player on battlefield
   - If equipment was attached to another creature, remove from old target's attachments
   - Set equipment.attached_to = target_id, target.attachments.push(equipment_id)
   - Return WaitingFor::Priority { player }
6. Add match arm for activating equip ability from Priority state: check if the activated ability is an equip ability (api_type contains "Equip" or ability text has "Equip"), validate sorcery-speed timing, then compute valid creature targets and return WaitingFor::EquipTarget.
7. Add `pub mod attach;` to effects/mod.rs.</action>

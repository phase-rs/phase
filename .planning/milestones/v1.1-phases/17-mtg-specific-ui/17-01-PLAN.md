Create three new utility modules:

**buttonStyles.ts** — Port from Alchemy (src/components/ui/buttonStyles.ts). Export `gameButtonClass` helper that accepts `{ tone, size?, disabled?, className? }` and returns Tailwind class string. Tones: neutral, emerald, amber, blue, red, indigo, slate. Sizes: xs, sm, md (default), lg. Base classes include `min-h-11 border border-solid font-semibold backdrop-blur-sm transition-colors duration-150 cursor-pointer focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-white/40 inline-flex items-center justify-center`. Disabled state adds `opacity-40 pointer-events-none`. Size maps to padding/text-size (xs: `px-2 py-1 text-xs rounded`, sm: `px-3 py-1.5 text-sm rounded-md`, md: `px-4 py-2 text-sm rounded-lg`, lg: `px-6 py-2.5 text-base rounded-lg`). Export types `GameButtonTone` and `GameButtonSize`.

**usePhaseInfo.ts** — Full rewrite for MTG's 12-phase turn structure. Export `usePhaseInfo()` hook that reads `gameStore.gameState.phase`. Returns `PhaseInfo` with:
- `displayKey`: one of 'draw' | 'main1' | 'combat' | 'main2' | 'end' (groups Untap/Upkeep/Draw as 'draw', all combat sub-phases as 'combat', End/Cleanup as 'end')
- `currentOrder`: number (0-4) for strip highlighting
- `phaseLabel`: human-readable label for current specific phase (e.g. "Declare Attackers" not "Combat")
- `phases`: Array of `{ key, label, order }` for the simplified strip display: `[{ key: 'draw', label: 'Draw', order: 0 }, { key: 'main1', label: 'Main 1', order: 1 }, { key: 'combat', label: 'Combat', order: 2 }, { key: 'main2', label: 'Main 2', order: 3 }, { key: 'end', label: 'End', order: 4 }]`
- `advanceLabel`: contextual label for advance/pass button ("Battle!" during Main 1, "End Turn" during Main 2, "Done" otherwise, "Resolve" when stack has items)
- `isCombatPhase`: boolean (true for BeginCombat through EndCombat)

Use module-level constants for the phases array (Zustand selector stability pattern). Hook reads phase from gameStore and stack length.

**boardSizing.ts** — Port Alchemy's container-aware card sizing pattern. Export `getCardSize(containerWidth: number, cardCount: number, options?)` that returns `{ width, height, gap }`. Uses MTG card aspect ratio (63/88 = 0.716). Default base width ~100px, min width 50px. Scale factor: `Math.min(1, containerWidth / (cardCount * (baseWidth + gap)))`. Also export `getStackCardSize(stackCount: number)` for stack card sizing (see RESEARCH.md pattern: base 160px, min 80px, shrink curve from 2+ items).

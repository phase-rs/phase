  <name>Task 1: Create BlockAssignmentLines with animated SVG</name>
  <files>client/src/components/board/BlockAssignmentLines.tsx</files>
  <action>
Port Alchemy's BlockAssignmentLines (201 lines) adapted for Forge.rs patterns. This replaces BlockerArrow.

**Architecture:** Render as a full-screen SVG overlay using `createPortal(svg, document.body)`. The SVG is `pointer-events-none fixed inset-0 z-30 w-full h-full`.

**Position tracking:** Use RAF polling pattern from Alchemy:
- Store positions in `useRef<Map<string, DOMRect>>()`
- On each animation frame, query `document.querySelector([data-object-id="${id}"])` for each blocker and attacker ID
- Compare with previous frame positions — count consecutive identical frames
- After 10 consecutive stable frames (positions didn't change), stop polling and set final positions
- Restart polling when `blockerAssignments` changes

**Line rendering:** For each blocker->attacker pair in `blockerAssignments`:
- Calculate center points from the stored DOMRects
- Render an SVG `<line>` from blocker center to attacker center
- Style: `strokeDasharray="8 4"` for dashed line, `stroke="rgba(249,115,22,0.7)"`, `strokeWidth={2.5}`

**Glow effect:** Add SVG `<filter>` with `<feGaussianBlur>` for glow:
```xml
<filter id="block-line-glow">
  <feGaussianBlur stdDeviation="3" result="blur" />
  <feMerge>
    <feMergeNode in="blur" />
    <feMergeNode in="SourceGraphic" />
  </feMerge>
</filter>
```
Apply `filter="url(#block-line-glow)"` to lines.

**Pulse dots:** For each line, render a small circle that animates along the line path. Use `animationDuration` based on line length. Implement with CSS animation or a `<circle>` that updates position via `useEffect` interval, moving from blocker to attacker center.

**Minimal VFX mode:** Check `preferencesStore.vfxQuality`. If "minimal", render simple solid lines without glow or pulse dots (like current BlockerArrow but positioned via the RAF pattern).

Read `blockerAssignments` from `uiStore`. Also read `gameState.combat?.blocker_to_attacker` for confirmed blocker assignments (after DeclareBlockers is submitted).

Component should render when `combatMode === 'blockers'` OR when `gameState.combat` has blocker assignments (during combat damage phase).
